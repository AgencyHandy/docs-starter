Create Client এন্ডপয়েন্ট আপনাকে একটি বাহ্যিক সিস্টেম থেকে সরাসরি আপনার AgencyHandy ওয়ার্কস্পেসে ক্লায়েন্ট যোগ করতে দেয়। লিডের বিপরীতে, ক্লায়েন্টরা হল সম্পূর্ণরূপে অনবোর্ড করা যোগাযোগ যাদের অর্ডার, ইনভয়েস এবং প্রকল্পে নিয়োগ দেওয়া যায়। CRM, অনবোর্ডিং ফর্ম, বা অন্য যেকোনো ডেটা উৎস থেকে নতুন ক্লায়েন্ট সিঙ্ক করতে এই এন্ডপয়েন্ট ব্যবহার করুন।

<Note>
  এই এন্ডপয়েন্ট ব্যবহার করার আগে, আপনার API কী এবং Company ID পেতে [শুরু করা](/bn/api/getting-started) গাইড সম্পন্ন করুন। Create Lead এন্ডপয়েন্টের বিপরীতে, ক্লায়েন্ট তৈরিতে আলাদা Role ID লুকআপ ধাপের **প্রয়োজন নেই**।
</Note>

## পূর্বশর্তসমূহ

- ✅ **Workspace Config → API Key** থেকে API কী তৈরি করা হয়েছে
- ✅ `GET {{URL}}/accounts/companies` থেকে Company ID পুনরুদ্ধার করা হয়েছে

## এন্ডপয়েন্ট

```
POST {{URL}}/members/bulk-client
```

## হেডার

| হেডার | মান |
|---|---|
| `x-api-key` | আপনার API কী |
| `companyId` | আপনার Company ID |
| `Content-Type` | `application/json` |

## অনুরোধের বডি

অনুরোধের বডি একটি **JSON অ্যারে**। আপনি একটি অনুরোধে এক বা একাধিক ক্লায়েন্ট তৈরি করতে পারেন।

<ParamField body="firstName" type="string" required>
  ক্লায়েন্টের প্রথম নাম।
</ParamField>

<ParamField body="lastName" type="string" required>
  ক্লায়েন্টের শেষ নাম।
</ParamField>

<ParamField body="email" type="string" required>
  ক্লায়েন্টের ইমেইল ঠিকানা। আপনার ওয়ার্কস্পেসে অনন্য হতে হবে।
</ParamField>

<ParamField body="isConvertedClient" type="boolean" required>
  একেবারে নতুন ক্লায়েন্টের জন্য `false` সেট করুন। বিদ্যমান লিডকে ক্লায়েন্টে রূপান্তর করার সময় `true` সেট করুন।
</ParamField>

<ParamField body="status" type="string">
  ক্লায়েন্টের স্ট্যাটাস। সাধারণ মান: `New`, `Active`। বাদ দিলে `New`-তে ডিফল্ট হয়।
</ParamField>

<ParamField body="contactNo" type="string">
  ক্লায়েন্টের ফোন নম্বর।
</ParamField>

<ParamField body="source" type="string">
  আপনি কীভাবে এই ক্লায়েন্ট পেয়েছেন। উদাহরণ মান: `website`, `referral`, `social`।
</ParamField>

<ParamField body="positionInBoard" type="number">
  বোর্ড কলামের মধ্যে ক্লায়েন্টের অবস্থান। `1`-এ ডিফল্ট হয়।
</ParamField>

## উদাহরণ অনুরোধ

```bash cURL
curl --request POST "https://api.agencyhandy.com/members/bulk-client" \
  --header "x-api-key: <YOUR_API_KEY>" \
  --header "companyId: <YOUR_COMPANY_ID>" \
  --header "Content-Type: application/json" \
  --data '[
    {
      "firstName": "John",
      "lastName": "Doe",
      "email": "john.doe@example.com",
      "isConvertedClient": false,
      "status": "New",
      "contactNo": "1234567890",
      "source": "website",
      "positionInBoard": 1
    }
  ]'
```

## সফলতার প্রতিক্রিয়া

```json
{
  "message": "Client created successfully",
  "createdMembers": [
    {
      "_id": "NEW_MEMBER_ID",
      "name": "John Doe",
      "status": "New",
      "role": "client"
    }
  ]
}
```

<ResponseField name="message" type="string">
  সফলতায় নিশ্চিতকরণ স্ট্রিং।
</ResponseField>

<ResponseField name="createdMembers" type="array">
  তৈরি ক্লায়েন্ট অবজেক্টের অ্যারে।
</ResponseField>

<ResponseField name="createdMembers[].\_id" type="string">
  নতুন তৈরি ক্লায়েন্টের অনন্য ID। API-এর মাধ্যমে ক্লায়েন্টকে অর্ডার বা ইনভয়েসে নিয়োগ দেওয়ার সময় এই ID ব্যবহার করুন।
</ResponseField>

<ResponseField name="createdMembers[].name" type="string">
  ক্লায়েন্টের পূর্ণ নাম (firstName + lastName)।
</ResponseField>

<ResponseField name="createdMembers[].status" type="string">
  সংরক্ষিত ক্লায়েন্টের স্ট্যাটাস।
</ResponseField>

<ResponseField name="createdMembers[].role" type="string">
  সদস্যকে বরাদ্দ ভূমিকা — `"client"` হবে।
</ResponseField>

<Note>
  এই এন্ডপয়েন্টের `companyId` হেডার `Id`-তে বড় হাতের **I** ব্যবহার করে — `companyId` — কিছু অন্য এন্ডপয়েন্টের বিপরীতে যেগুলি `companyid` (সম্পূর্ণ ছোট হাতের) ব্যবহার করে। প্রমাণীকরণ ত্রুটি এড়াতে উপরে দেখানো সঠিক কেসিং ব্যবহার করুন।
</Note>

<Tip>
  একটি একক API কলে বাল্ক-তৈরি ক্লায়েন্টের জন্য অ্যারেতে একাধিক ক্লায়েন্ট অবজেক্ট পাস করুন। প্রতিটি অবজেক্টে একটি অনন্য `email` ঠিকানা থাকতে হবে। ডুপ্লিকেট ইমেইল সেই এন্ট্রির জন্য একটি যাচাইকরণ ত্রুটি ঘটাবে।
</Tip>
