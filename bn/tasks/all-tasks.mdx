AgencyHandy-তে All Tasks পৃষ্ঠা আপনাকে আপনার এজেন্সির সক্রিয় কার্যপ্রবাহ জুড়ে প্রতিটি টাস্কের একটি একক, কেন্দ্রীভূত ভিউ দেয়। পৃথক অর্ডারে খোঁজাখুঁজির পরিবর্তে, আপনি একটি জায়গায় সমস্ত প্রাসঙ্গিক টাস্ক দেখতে পারবেন, নিয়োগকৃত ব্যক্তি, অগ্রাধিকার বা তারিখ দিয়ে সেগুলো ফিল্টার করতে পারবেন এবং তালিকা থেকে সরাসরি সম্পাদনা করতে পারবেন। দৃশ্যমানতা স্বয়ংক্রিয়ভাবে আপনার ভূমিকা অনুসারে সীমাবদ্ধ — Super Admins সবকিছু দেখেন, Managers তাদের নিযুক্ত অর্ডার দেখেন এবং Employees শুধুমাত্র তাদের নিজের টাস্ক দেখেন।

## All Tasks পৃষ্ঠায় অ্যাক্সেস করা

1. আপনার AgencyHandy অ্যাকাউন্টে লগইন করুন।
2. বাম সাইডবারে **Tasks** ক্লিক করুন।
3. কেন্দ্রীভূত টাস্ক ড্যাশবোর্ড খুলতে **All Tasks** নির্বাচন করুন।

<Note>
  All Tasks পৃষ্ঠা শুধুমাত্র **Ongoing** বা **In Review** স্ট্যাটাসে থাকা অর্ডারের টাস্কগুলো প্রদর্শন করে। অন্য যেকোনো অবস্থায় (যেমন, Completed, Pending) অর্ডারের সাথে যুক্ত টাস্কগুলো এখানে প্রদর্শিত হয় না।
</Note>

## ভূমিকা অনুসারে টাস্ক দৃশ্যমানতা

All Tasks পৃষ্ঠায় আপনি কী দেখবেন তা সম্পূর্ণভাবে আপনার ভূমিকার উপর নির্ভর করে।

| ভূমিকা | তারা কী দেখবেন |
|------|---------------|
| **Super Admin / Admin** | প্রতিটি অর্ডার, টিম এবং স্ট্যাটাস জুড়ে সমস্ত টাস্ক |
| **Project Manager (PM)** | শুধুমাত্র যেসব অর্ডারে তারা Manager হিসেবে নিযুক্ত সেগুলোর টাস্ক |
| **Employee / Assignee** | শুধুমাত্র সরাসরি তাদের কাছে নিযুক্ত টাস্ক — এমনকি যদি সতীর্থরা একই অর্ডারে কাজ করছেন তবুও |

<Tip>
  যদি একজন PM কোনো টাস্ক দেখতে না পান যা তিনি আশা করছিলেন, নিশ্চিত করুন যে তাদের প্রাসঙ্গিক অর্ডারে Manager হিসেবে নিযুক্ত করা হয়েছে। PM-রা সেই অর্ডারের নিযুক্ত Manager না হলে টাস্ক দেখতে পান না।
</Tip>

## অনুসন্ধান, ফিল্টার এবং বাছাই

All Tasks পৃষ্ঠায় আপনার প্রয়োজনীয় বিষয়টি ঠিক খুঁজে পেতে সাহায্য করার জন্য নিয়ন্ত্রণের একটি সম্পূর্ণ সেট রয়েছে।

### অনুসন্ধান

নাম, নিয়োগকৃত ব্যক্তি বা সম্পর্কিত অর্ডার দিয়ে টাস্ক খুঁজে পেতে অনুসন্ধান বার ব্যবহার করুন। আপনি টাইপ করার সাথে সাথে ফলাফল রিয়েল টাইমে আপডেট হয়।

### ফিল্টার

ফিল্টার প্যানেল খুলতে **Filter → + Add Filter** ক্লিক করুন। সুনির্দিষ্ট ফলাফলের জন্য আপনি একাধিক ফিল্টার একত্রিত করতে পারবেন:

| ফিল্টার | এটি কী করে |
|--------|--------------|
| **Order ID** | নির্দিষ্ট অর্ডার নম্বরের সাথে লিঙ্কড টাস্ক দেখান |
| **Assignee** | নির্দিষ্ট টিম সদস্যকে নিযুক্ত টাস্ক দেখান |
| **Status** | টাস্কের অগ্রগতি অনুসারে ফিল্টার করুন: Pending, In Progress, Review, Done ইত্যাদি |
| **Priority** | High, Medium বা Low অগ্রাধিকারের টাস্কে মনোযোগ দিন |
| **Label** | টাস্ক লেবেল বা বিভাগ দ্বারা সীমাবদ্ধ করুন |
| **Kick Off Date** | নির্দিষ্ট তারিখে বা তার পরে শুরু হওয়া টাস্ক দেখান |
| **End Date** | নির্দিষ্ট তারিখের মধ্যে শেষ হওয়া টাস্ক দেখান |

<Note>
  ফিল্টারগুলো শুধুমাত্র All Tasks পৃষ্ঠায় পাওয়া যায়। এগুলো আপনার ব্রাউজার সেশনে অব্যাহত থাকে কিন্তু পৃষ্ঠা রিফ্রেশ করলে রিসেট হয়। ফিল্টারগুলো শুধুমাত্র আপনার ভিউকে প্রভাবিত করে এবং অন্য ব্যবহারকারীরা কী দেখেন তা পরিবর্তন করে না।
</Note>

সমস্ত ফিল্টার সাফ করতে, সম্পূর্ণ, আনফিল্টারড তালিকায় ফিরতে **Clear Filters** ক্লিক করুন।

### বাছাই

টাস্ক তালিকা বাছাই করুন:
- **Due date** — আসন্ন বা ইতিমধ্যে মেয়াদোত্তীর্ণ টাস্কগুলো উপরে আনুন
- **Priority** — সবচেয়ে জটিল টাস্কগুলো শীর্ষে আনুন
- **Creation time** — সম্প্রতি যোগ করা টাস্কগুলো প্রথমে দেখুন

### ব্রেডক্রাম্ব নেভিগেশন

সাবটাস্ক, টাস্ক তালিকা এবং প্যারেন্ট অর্ডারের মধ্যে নেভিগেট করার সময় প্রেক্ষাপট বজায় রাখতে পৃষ্ঠার শীর্ষে ব্রেডক্রাম্ব ট্রেইল ব্যবহার করুন। একটি নির্দিষ্ট টাস্কে ড্রিল করার পরে এবং বিস্তৃত তালিকায় ফিরে আসার প্রয়োজন হলে এটি বিশেষভাবে সহায়ক।

  </Tab>

## টাস্ক মুছে ফেলার নিয়ম

All Tasks পৃষ্ঠার টাস্কগুলো সরাসরি তাদের প্যারেন্ট অর্ডারের সাথে যুক্ত।

- যখন একটি **অর্ডার মুছে ফেলা হয়**, সেই অর্ডারের সাথে সম্পর্কিত সমস্ত টাস্ক All Tasks পৃষ্ঠা থেকে **স্বয়ংক্রিয়ভাবে এবং স্থায়ীভাবে সরিয়ে দেওয়া হয়**।
- এই মুছে ফেলা সমস্ত ব্যবহারকারীর ভূমিকাকে প্রভাবিত করে — অর্ডার মুছে ফেলার পরে কেউ সেই টাস্কগুলো দেখতে পারবেন না।
- অর্ডার মুছে ফেলার মাধ্যমে মুছে যাওয়া টাস্কের জন্য **কোনো পুনরুদ্ধারের বিকল্প নেই**।

<Warning>
  একটি অর্ডার মুছে ফেলা এর সমস্ত টাস্ক স্থায়ীভাবে সরিয়ে দেয়। অর্ডার মুছে ফেলার আগে নিশ্চিত করুন যে গুরুত্বপূর্ণ টাস্ক ইতিহাস বা ডেলিভারেবল রেকর্ড অন্যত্র সংরক্ষিত আছে।
</Warning>
