オーダーフォームは、ウェブサイトの訪問者やリードを有料クライアントに転換する最速の方法です。別々のサービス見積もりを送って支払いリンクを追いかける代わりに、サービスカタログ、オプションのアップセル、プロジェクト入力、チェックアウトを組み合わせた1つのフォームを作成します。これにより、クライアントは1つの流れで閲覧、カスタマイズ、支払いを完了できます。公開後、フォームは直接送信したり各クライアントのポータル内に表示したりできる共有可能なパブリックURLで利用できます。

## オーダーフォームに含められるもの

<CardGroup cols={2}>
  <Card title="サービスとアップセル" icon="cart-shopping">
    コアサービス、オプションのアップセルパッケージ、アドオンを追加して、クライアントが往復の会話なしに自分でオーダーを組み立てられるようにします。
  </Card>
  <Card title="プロジェクト入力フィールド" icon="file-lines">
    クライアントがチェックアウト前に、オーダーフォーム上で直接、書面のブリーフ、締め切り、ファイルのアップロード、スプレッドシートなどを収集します。
  </Card>
  <Card title="利用規約" icon="file-contract">
    購入フローの一部として、クライアントにサービス規約の確認と同意を求め、エージェンシーを法的に保護します。
  </Card>
  <Card title="デジタル署名" icon="signature">
    オーダーが確定する前に、クライアントが正式に同意を確認できるよう署名ブロックを追加します。
  </Card>
</CardGroup>

## オーダーフォームを作成する

<Steps>
  <Step title="フォームセクションを開く">
    左側のサイドバーで **フォーム** をクリックし、リストから **オーダーフォーム** を選択します。
  </Step>
  <Step title="新しいフォームを作成する">
    右上の **新しいオーダーを作成** をクリックします。空のフォームビルダーが開きます。
  </Step>
  <Step title="サービスとアップセルを追加する">
    フォームビルダーの **サービス** セクションで、**サービスを追加** をクリックして、事前設定済みのサービスを1つ以上追加します。各サービスに対して以下の操作もできます：

    - **アップセルを追加** をクリックして、クライアントがオプトインできるアップグレードや補完的なパッケージを提供します。
    - **アドオン** をクリックして、延長サポートや追加成果物などのオプションのエクストラを含めます。

    <Note>
      サービス、アップセル、アドオンは、オーダーフォームに添付する前に **サービス** カタログで作成しておく必要があります。
    </Note>
  </Step>
  <Step title="プロジェクト入力フィールドを追加する">
    **プロジェクト入力** セクションで、任意のフィールドタイプをクリックしてフォームに追加します。利用可能なフィールドタイプは以下の通りです：

    | フィールドタイプ | 最適な用途 |
    |---|---|
    | **短い回答**（120文字） | プロジェクトタイトル、クイックリファレンス |
    | **段落** | オープンエンドのブリーフと説明 |
    | **書式付きテキスト** | リッチフォーマット付きの詳細仕様 |
    | **複数選択（チェックボックス）** | 該当する複数のオプションを選択 |
    | **選択（ラジオボタン）** | 排他的な単一オプションの選択 |
    | **日付** | 締め切り、公開日、イベント日 |
    | **ファイル添付** | デザインブリーフ、参考素材、ブランドファイル |
    | **スプレッドシート** | 一括データ、キーワードリスト、商品カタログ |

    フィールドを上下にドラッグして並び替えます。フィールドに **必須** を設定すると、そのフィールドなしでは送信できなくなります。
  </Step>
  <Step title="ユーティリティフィールドを設定する">
    **ユーティリティ** セクションで、クライアントが購入を完了する前にポリシーに正式に同意するよう、**利用規約** ブロックや **署名** フィールドを追加します。
  </Step>
  <Step title="プレビューと公開">
    **フォームのプレビュー** をクリックして、クライアントに見える内容を確認します。満足したら、**保存して公開** をクリックします。フォームが公開され、共有できる状態になります。
  </Step>
</Steps>

## オーダーフォームを共有する

公開後、クライアントにフォームを見てもらう主な方法が2つあります。

<Tabs>
  <Tab title="直接リンク">
    オーダーフォームの詳細ページからパブリックフォームのURLをコピーし、メール、チャット、またはウェブサイトを通じてクライアントに送信します。クライアントがフォームにアクセスまたは送信するためのログインは不要です。

    <Tip>
      リンクをメールの署名やウェブサイトの「始める」ボタンに貼り付けることで、新しいリードがいつでも自己サービスできるようになります。
    </Tip>
  </Tab>
  <Tab title="クライアントポータル">
    外部リンクの共有なしに、クライアントのポータル内に直接オーダーフォームを表示させることができます。クライアントの **フォーム** タブに表示されます。以下の手順を参照してください。
  </Tab>
</Tabs>

### クライアントポータルにオーダーフォームを表示する

<Steps>
  <Step title="フォームの共有設定を開く">
    **フォーム → オーダーフォーム** に移動し、表示したいフォームを見つけ、**三点メニュー（⋯）** をクリックして、**フォームを共有** を選択します。
  </Step>
  <Step title="ポータルの表示を有効にする">
    **パブリックフォームを共有** モーダルで、**クライアントのポータルにこのフォームを表示する** を有効にします。
  </Step>
  <Step title="送信ルールを設定する">
    クライアントがフォームを複数回オーダーまたは再送信できるようにしたい場合は、オプションで **1人のクライアントが複数回回答を送信できる** を有効にします。
  </Step>
  <Step title="クライアントリストを確認する">
    **共有先** リストを確認して、正しいクライアントが含まれていることを確かめます。そのクライアントだけがポータルでフォームを見ることができます。
  </Step>
  <Step title="保存">
    **送信** をクリック（クライアントにメールでも通知される）するか、モーダルを閉じます。表示設定は自動的に保存されます。
  </Step>
</Steps>

<Note>
  有効にすると、リストされた各クライアントのポータルの **フォーム** タブにフォームが表示されます。ポータルを通じて送信するにはクライアントのログインが必要ですが、パブリックリンクからの送信にはログインは不要です。
</Note>

## クライアントのオーダーフォーム体験

クライアントがオーダーフォームのリンクを開いたり、ポータルでアクセスすると、以下の流れになります：

1. 利用可能なサービスを閲覧し、必要なものを選択します。
2. 含めたいアップセルやアドオンを選びます。
3. プロジェクト入力フィールドに要件を入力します。
4. 利用規約を確認して同意します（含まれている場合）。
5. デジタル署名を追加します（必要な場合）。
6. 支払いを完了してオーダーを確定します。

送信されたすべてのデータはAgencyHandyに即座に取り込まれ、新しいオーダーがダッシュボードに自動的に作成されます。

## 重要な注意事項

<Warning>
  クライアントはフォームを送信する前に、すべての **必須** フィールドを入力する必要があります。クライアントが必要な情報を正確に理解できるよう、すべてのフィールドに明確なラベルを付けてください。
</Warning>

- フィールドラベルは簡潔で専門用語を避けてください。クライアントは社内の用語に馴染みがない場合があります。
- 利用規約と署名フィールドはエージェンシーに法的保護を提供します。有料のオーダーフォームには含めることをお勧めします。
- 公開済みのフォームはいつでも編集できます。保存後にリンクを開いたユーザーには変更が即座に反映されます。
