AgencyHandyのすべてのタスクページでは、エージェンシー全体のアクティブなワークフローに属するすべてのタスクを1か所で一元表示できます。個々のオーダーを掘り下げる手間なく、関連するすべてのタスクを1か所で確認し、担当者、優先度、日付でフィルタリングし、リストから直接編集できます。表示内容は役割に基づいて自動的に範囲が設定されます — スーパー管理者はすべてを閲覧でき、マネージャーは割り当てられたオーダーを閲覧でき、従業員は自分のタスクのみ閲覧できます。

## すべてのタスクページへのアクセス

1. AgencyHandyアカウントにログインします。
2. 左サイドバーの**タスク**をクリックします。
3. **すべてのタスク**を選択して一元タスクダッシュボードを開きます。

<Note>
  すべてのタスクページには、**進行中**または**レビュー中**ステータスのオーダーのタスクのみが表示されます。その他のステータス（完了、保留中など）のオーダーに紐付いたタスクはここには表示されません。
</Note>

## 役割別のタスク表示

すべてのタスクページで表示される内容は、役割によって完全に異なります。

| 役割 | 表示される内容 |
|------|---------------|
| **スーパー管理者 / 管理者** | すべてのオーダー、チーム、ステータスにわたるすべてのタスク |
| **プロジェクトマネージャー（PM）** | マネージャーとして割り当てられたオーダーのタスクのみ |
| **従業員 / 担当者** | 直接割り当てられたタスクのみ — 同じオーダーでチームメイトが作業していても |

<Tip>
  PMが確認できるはずのタスクが見つからない場合は、該当するオーダーのマネージャーとして割り当てられているかどうかを確認してください。PMはそのオーダーの割り当てマネージャーでない限りタスクを閲覧できません。
</Tip>

## タスクがこのページに表示される条件

以下の条件が満たされたとき、タスクがすべてのタスクページに表示されます：

<CardGroup cols={2}>
  <Card title="タスク表示をトリガーするオーダー" icon="circle-check">
    - 手動オーダー（スポットまたはサブスクリプション）が作成される
    - Stripeまたは手動支払い方法によるカタログオーダーが発注される
    - エージェンシーによって提案書が承認される
  </Card>
  <Card title="タスクを非表示にするオーダー" icon="eye-slash">
    - エージェンシーがまだ承認していないクライアントからのタスクリクエスト
    - エージェンシーがまだ承認していないトライアルオーダー
    - 手動銀行振込確認待ちの保留中ステータスのオーダー
  </Card>
</CardGroup>

## 検索、フィルタリング、ソート

すべてのタスクページには、必要なものを正確に見つけるための完全なコントロールセットが含まれています。

### 検索

検索バーを使用して名前、担当者、または関連オーダーでタスクを検索します。入力するにつれて結果がリアルタイムで更新されます。

### フィルター

**フィルター → + フィルターを追加**をクリックしてフィルターパネルを開きます。複数のフィルターを組み合わせて精密な結果を得ることができます：

| フィルター | 機能 |
|--------|--------------|
| **オーダーID** | 特定のオーダー番号に紐付いたタスクを表示 |
| **担当者** | 特定のチームメンバーに割り当てられたタスクを表示 |
| **ステータス** | タスクの進捗でフィルタリング：保留中、進行中、レビュー中、完了など |
| **優先度** | 高、中、低の優先度タスクに集中 |
| **ラベル** | タスクのラベルまたはカテゴリで絞り込み |
| **開始日** | 特定の日付以降に開始するタスクを表示 |
| **終了日** | 特定の日付までに完了するタスクを表示 |

<Note>
  フィルターはすべてのタスクページでのみ利用可能です。ブラウザセッション中は維持されますが、ページ更新時にリセットされます。フィルターは自分の表示にのみ影響し、他のユーザーの表示は変更されません。
</Note>

すべてのフィルターをクリアするには、**フィルターをクリア**をクリックしてフィルタリングされていない完全なリストに戻ります。

### ソート

タスクリストのソート方法：
- **納期** — 近づいているか既に期限超過のタスクを表示
- **優先度** — 最も重要なタスクを上位に表示
- **作成時刻** — 最近追加されたタスクを最初に表示

### パンくずナビゲーション

ページ上部のパンくずリストを使用して、サブタスク、タスクリスト、親オーダー間のナビゲーション中にコンテキストを維持します。特定のタスクに掘り下げた後、より広いリストに戻る必要がある場合に特に役立ちます。

## タスクの編集

すべてのタスクページでタスクを変更するには2つの方法があります：

<Tabs>
  <Tab title="インライン編集">
    タスク行内の編集可能なフィールド（ステータス、担当者、納期など）を直接クリックして、リストビューを離れることなく変更します。これは素早い更新のための最速オプションです。
  </Tab>
  <Tab title="モーダル編集">
    タスク名をクリックして完全なタスク詳細モーダルを開きます。ここからすべてのタスクプロパティを更新し、メモを追加し、サブタスクを表示し、完全なタスク履歴を確認できます。より徹底的な編集やコンテキストが必要な場合に使用します。
  </Tab>
</Tabs>

## タスクの削除ルール

すべてのタスクページのタスクは、親オーダーに直接紐付いています。

- **オーダーが削除される**と、そのオーダーに関連するすべてのタスクが**すべてのタスクページから自動的かつ永久に削除**されます。
- この削除はすべてのユーザー役割に影響します — オーダーが削除されるとすべてのユーザーがそれらのタスクを閲覧できなくなります。
- オーダー削除によって削除されたタスクには**復元オプションがありません**。

<Warning>
  オーダーを削除すると、そのすべてのタスクが永久に削除されます。オーダーを削除する前に、重要なタスク履歴や成果物の記録が他の場所に保存されていることを確認してください。
</Warning>

## 追加の注意事項

<Accordion title="リアルタイム更新">
  オーダー、タスクの割り当て、またはタスクのステータスへの変更は、すべてのタスクページにリアルタイムで反映されます。チームメイトがページを開いている状態でタスクを更新すると、更新なしに変更が確認できます。
</Accordion>

<Accordion title="クライアントからのタスクリクエスト">
  クライアントが送信したタスクは、エージェンシーが明示的に承認するまですべてのタスクページに表示されません。承認されるまで、これらのタスクは保留状態のままで、一元表示には表示されません。
</Accordion>

<Accordion title="タスクとオーダーステータス">
  タスクは進行中またはレビュー中ステータスのオーダーのみに表示されます。オーダーが完了に移行するか一時停止されると、そのタスクはここには表示されなくなります。これらのステータス以外のオーダーに紐付いたタスクを表示するには、特定のオーダーに移動してください。
</Accordion>
