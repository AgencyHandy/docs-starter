AgencyHandy의 Slack 연동을 통해 팀원들이 앱을 전환하지 않고도 최신 정보를 파악할 수 있습니다. 연결되면 AgencyHandy는 태스크, 주문, 또는 기타 워크스페이스 활동이 변경될 때마다 선택한 Slack 채널에 자동 알림을 게시합니다 — 팀이 항상 실시간으로 상황을 파악할 수 있습니다.

<Note>
  Slack 연동의 연결 및 관리는 **슈퍼관리자**, **관리자**, 그리고 연동 권한이 있는 **매니저** 역할만 가능합니다. 연결을 승인하려면 Slack 워크스페이스 관리자 권한이 필요합니다.
</Note>

## Slack 연결

<Steps>
  <Step title="연동 열기">
    왼쪽 사이드바에서 **연동**을 클릭합니다.
  </Step>

  <Step title="Slack 카드 찾기">
    **Slack** 카드로 스크롤하여 **연결**을 클릭합니다.
  </Step>

  <Step title="Slack에서 AgencyHandy 권한 부여">
    Slack 권한 부여 창이 열립니다. Slack 워크스페이스에 아직 로그인하지 않은 경우 지금 로그인합니다. AgencyHandy가 요청하는 권한을 검토하고 **허용**을 클릭하여 접근 권한을 부여합니다.
  </Step>

  <Step title="알림 채널 선택">
    권한 부여 후 AgencyHandy 알림을 받을 Slack 채널을 선택합니다. 드롭다운에서 채널을 선택하고 **저장**을 클릭하여 확인합니다.
  </Step>

  <Step title="설정 확인">
    Slack 카드가 **연결됨**으로 업데이트됩니다. AgencyHandy가 즉시 선택한 채널에 알림을 게시하기 시작합니다.
  </Step>
</Steps>

## Slack으로 전송되는 알림

AgencyHandy는 다음 이벤트가 발생할 때마다 Slack 메시지를 게시합니다:

| 카테고리 | 알림을 트리거하는 이벤트 |
|---|---|
| **태스크** | 태스크 생성, 업데이트, 배정, 또는 완료 |
| **주문** | 주문 생성 또는 상태 변경 |
| **댓글** | 태스크 또는 주문에 새 댓글 추가 |
| **파일 첨부** | 태스크에 파일 또는 첨부 파일 추가 |
| **티켓** | 티켓 생성, 배정, 또는 상태 변경 |

각 Slack 메시지에는 다음이 포함됩니다:

- 태스크, 주문, 또는 티켓의 **이름**
- 메시지를 트리거한 **상태 변경** 또는 작업
- AgencyHandy의 관련 항목으로 돌아가는 **링크**

태스크에 파일이나 첨부 파일이 추가되면 Slack 메시지에 해당 첨부 파일의 미리보기도 포함됩니다.

<Tip>
  알림을 체계적으로 관리하고 주요 팀 채널이 복잡해지지 않도록 전용 **#agency-handy-alerts** 채널(또는 유사한 이름)을 선택하세요.
</Tip>

## Slack 연결 해제

알림을 중지하려면 **연동**으로 돌아가 Slack 카드에서 **연결 해제**를 클릭합니다. 모든 알림이 즉시 중지됩니다.

## 중요 참고 사항

- **연결당 하나의 워크스페이스** — 각 AgencyHandy 워크스페이스에는 단일 Slack 워크스페이스만 연결할 수 있습니다. 다른 Slack 워크스페이스로 전환하려면 현재 연결을 해제하고 다시 연결합니다.
- **채널 가시성** — 알림은 선택한 Slack 채널의 모든 사람에게 표시됩니다. 공유되는 정보 수준에 적합한 채널인지 확인하세요 (예: 클라이언트에 민감한 주문 세부 정보를 공개 채널에 게시하지 않도록 주의).
- **재승인** — Slack 토큰이 만료되거나 권한이 변경되면 연동으로 돌아가 다시 연결하여 알림을 복원하세요.
