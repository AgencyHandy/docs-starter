태스크는 주문을 납품물로 전환하는 작업 단위입니다. 각 태스크는 특정 주문 안에 존재하며 팀 구성원에게 배정될 수 있고, 자체적인 상태, 일정, 첨부 파일, 피드백 스레드를 추적합니다. 이 페이지에서는 태스크를 생성하고 관리하는 방법, 클라이언트가 태스크를 요청하는 방법, 시간 추적 및 파일 피드백 작동 방식, 그리고 모든 주문에 걸쳐 태스크를 한 번에 보는 방법을 설명합니다.

## 역할 요구 사항

| 작업 | 수행 가능한 역할 |
|---|---|
| 태스크 생성 | 슈퍼 어드민, 어드민, 프로젝트 매니저 |
| 태스크 수정 | 슈퍼 어드민, 어드민, 프로젝트 매니저 |
| 태스크 삭제 | 슈퍼 어드민, 어드민, 프로젝트 매니저 |
| 검토 중/완료 태스크 보기 | 슈퍼 어드민, 어드민, PM, 클라이언트 |
| 피드백 전송 및 답변 | 슈퍼 어드민, 어드민, PM |
| 태스크에 시간 기록 | 배정된 모든 사용자 (슈퍼 어드민과 어드민은 다른 사람 대신 기록 가능) |
| 태스크 요청 (클라이언트) | 클라이언트 |
| 요청된 태스크 승인/거부 | 슈퍼 어드민, 어드민 |

<Note>
주문에 배정된 직원은 **In Review** 또는 **Done** 상태의 태스크를 볼 수 있습니다. **In Progress** 상태의 태스크는 직원에게 보이지 않습니다.
</Note>

## 태스크 생성

<Steps>
  <Step title="주문 열기">
    왼쪽 사이드바에서 **Orders**로 이동하여 태스크를 추가할 주문을 클릭합니다.
  </Step>

  <Step title="새 태스크 추가">
    주문 내에서 **Tasks** 섹션으로 스크롤하여 **+ Add New Task**를 클릭합니다. 태스크 생성 양식이 나타납니다.
  </Step>

  <Step title="태스크 세부 정보 입력">
    사용 가능한 필드를 작성합니다:

    | 필드 | 참고 사항 |
    |---|---|
    | **Task Name** | 작업 항목에 대한 명확하고 간결한 제목 |
    | **Description** | 상세한 맥락, 요구 사항 또는 지침 |
    | **Assignee** | 태스크 완료를 담당하는 팀 구성원 |
    | **Label** | 태스크 구성을 위한 분류 태그 (예: Design, Copy) |
    | **Start Date** | 태스크 작업 시작 일자 |
    | **Due Date** | 태스크 완료 마감 일자 |
    | **Client Review Required** | 태스크를 완료로 표시하기 전에 클라이언트의 승인을 요구하는 체크박스 — 기본적으로 선택됨 |
    | **Attachments** | 참고 파일 또는 지원 자료 |
  </Step>

  <Step title="태스크 저장">
    **Save**를 클릭하여 태스크를 생성합니다. 클라이언트는 새 태스크가 주문에 추가되었다는 자동 알림을 받습니다.
  </Step>
</Steps>

<Tip>
**Client Review Required** 체크박스는 모든 새 태스크에서 기본적으로 활성화되어 있습니다. 태스크가 순전히 내부용이고 클라이언트 확인이 필요 없는 경우, 저장 전에 체크를 해제할 수 있습니다.
</Tip>

## 태스크 상태

태스크는 다음 상태를 거쳐 진행됩니다:

- **In Progress** — 태스크가 진행 중입니다 (클라이언트나 직원에게 보이지 않음)
- **In Review** — 태스크가 검토를 위해 제출되었습니다; 클라이언트에게 보임
- **Done** — 태스크가 승인되어 완료되었습니다

<Note>
클라이언트 또는 검토자가 **In Review** 상태에서 태스크를 거부하면, 상태가 자동으로 **In Progress**로 돌아갑니다.
</Note>

## 태스크 모달 및 활동 페이지

태스크를 클릭하면 태스크의 활동 허브 역할을 하는 **태스크 모달** (전체 상세 보기)이 열립니다. 여기서 다음을 확인하고 관리할 수 있습니다:

- 현재 상태, 담당자, 시작 일자, 마감 일자
- 태스크 설명 및 첨부 파일
- 모든 피드백 스레드와 답변
- 태스크의 모든 업데이트, 댓글, 상태 변경을 보여주는 완전한 활동 로그

클라이언트 검토가 활성화된 경우 클라이언트를 포함하여 주문에 관여된 모든 사람이 가시성이 있는 태스크의 태스크 모달에 접근할 수 있습니다.

## 태스크 수정

태스크가 생성된 후 세부 정보를 업데이트하려면, 태스크 모달을 열고 관련 필드(이름, 설명, 담당자, 날짜, 레이블 또는 첨부 파일)를 수정합니다. 변경 사항을 저장하여 업데이트를 적용합니다. 슈퍼 어드민, 어드민, PM만 태스크를 수정할 수 있습니다.

## 태스크 삭제

태스크를 삭제하려면 태스크를 열고 태스크 작업 메뉴의 삭제 옵션을 사용합니다. 삭제는 영구적입니다 — 관련된 모든 시간 로그와 파일 버전이 태스크와 함께 제거됩니다.

<Warning>
태스크를 삭제하면 모든 파일 버전과 관련 피드백 댓글도 함께 삭제됩니다. 이 작업은 취소할 수 없습니다.
</Warning>

## 시간 추적

AgencyHandy에는 외부 도구 없이 팀이 태스크에 직접 시간을 기록할 수 있는 내장 시간 추적 기능이 포함되어 있습니다.

### 타이머 시작

<Steps>
  <Step title="태스크 열기">
    **Orders → Tasks**로 이동하여 시간을 추적할 태스크를 엽니다.
  </Step>

  <Step title="타이머 시작">
    **Start Timer**를 클릭합니다. 타이머가 실행되기 시작하고 태스크 이름, 실시간 경과 시간, 총 기록 시간이 표시됩니다.
  </Step>

  <Step title="완료 시 일시 정지 또는 중지">
    추적을 일시적으로 중단하려면 **Pause Timer**를, 항목을 완료하려면 **Stop Timer**를 클릭합니다. 중지하면 날짜, 시작 시간, 종료 시간, 총 소요 시간, 선택적 메모를 확인합니다.
  </Step>
</Steps>

<Note>
한 번에 하나의 활성 타이머만 실행할 수 있습니다. 새 타이머를 시작하면 기존 타이머가 자동으로 일시 정지됩니다.
</Note>

<Warning>
태스크가 **Done**으로 표시되면 타이머가 비활성화됩니다. 시간 기록을 다시 활성화하려면 태스크를 다시 엽니다.
</Warning>

### 수동 시간 입력

소급하여 시간을 기록해야 하는 경우, 태스크에서 **Manual Time Entry**를 클릭하고 다음을 입력합니다:

- **날짜**
- **시작 시간과 종료 시간** (또는 시간과 분 단위의 **소요 시간**)
- **메모** (선택 사항)

여러 사용자가 동일한 태스크에 동시에 시간을 기록할 수 있습니다. 슈퍼 어드민과 어드민은 모든 사용자를 대신하여 수동 시간 항목을 생성할 수 있습니다.

### 작업 로그

타이머 옆의 **Work Log** 버튼을 클릭하면 태스크의 모든 시간 항목에 대한 전체 내역(날짜, 시작/종료 시간, 총 활성 시간, 메모 포함)을 볼 수 있습니다. 어드민과 슈퍼 어드민은 모든 항목을 삭제할 수 있으며, 일반 사용자는 본인 항목만 삭제할 수 있습니다.

### 타임시트

왼쪽 사이드바에서 **Timesheets**로 이동하면 기록된 모든 시간을 주문 전체에 걸쳐 볼 수 있습니다. 태스크 이름, 주문 ID, 담당자, 또는 날짜 범위(일별, 주별, 월별)로 필터링할 수 있으며, 팀 구성원, 서비스 이름, 또는 태스크 이름별로 결과를 그룹화할 수 있습니다. 타임시트 보기에서 직접 타이머를 시작하거나 수동 항목을 추가할 수도 있습니다.

## 파일 피드백

AgencyHandy의 파일 피드백 시스템을 통해 귀하와 클라이언트는 태스크 내에서 직접 납품물에 정밀하고 시각적인 댓글을 남길 수 있습니다. 버전 관리와 스레드 토론이 내장되어 있으며, 이미지, PDF, 동영상, 라이브 웹사이트를 지원합니다.

### 검토용 파일 업로드

태스크를 열고 **Attachments** 섹션에서 **➕** 아이콘을 클릭하여 이미지, PDF, 동영상을 업로드하거나 라이브 웹사이트 URL을 붙여넣습니다.

### 파일 유형별 피드백 남기기

**이미지 및 PDF**
파일을 열고 주석 도구 모음(화살표, 선, 사각형, 강조 표시)을 사용하여 특정 영역에 마크업합니다. 영역을 선택하고 댓글을 입력한 후 팀, 클라이언트, 또는 둘 다에게 공유할지 선택합니다. 주석을 취소하거나 다시 실행하고 주석 색상을 변경할 수도 있습니다.

**동영상**
동영상을 열고 피드백이 적용되는 정확한 프레임에서 일시 정지합니다. 댓글을 입력하면 동영상이 자동으로 일시 정지됩니다. 피드백에는 해당 순간의 타임스탬프가 기록됩니다. 타임스탬프를 제거하여 일반 댓글을 남길 수도 있습니다.

**라이브 웹사이트**
**➕** 버튼을 사용하여 태스크에 사이트 URL을 추가한 후 AgencyHandy 브라우저 확장 프로그램을 설치합니다. **View Mode**(인터랙티브)로 사이트를 열거나 **Review Mode**로 전환하여 스크린샷을 찍고 그리기, 텍스트, 도형 도구로 주석을 달 수 있습니다.

### 댓글 작업

각 댓글에는 다음 옵션이 있는 세 점 메뉴가 있습니다:

- 댓글 수정
- 답글 복사
- 댓글 삭제
- 내부 팀에만 가시성 제한

### 피드백 해결

댓글의 **✓** 버튼을 클릭하여 **해결됨**으로 표시합니다. 해결된 댓글은 참고용으로 계속 표시되며 다시 열 수 있습니다. 귀하가 남긴 댓글이 다른 사람에 의해 해결되면 알림을 받습니다.

### 파일 버전

**Upload New Version**을 클릭하여 수정된 파일을 추가합니다. 이전 버전과 해당 피드백은 보존되며 버전 드롭다운에서 접근할 수 있습니다. 이전 버전을 열면 AgencyHandy가 더 최신 버전이 있음을 알려줍니다.

각 파일에 **Needs Review** 또는 **Approved** 레이블을 붙여 검토 상태를 한눈에 전달할 수 있습니다.

<Note>
태스크에 **Client Review Required** 체크박스가 활성화된 경우에만 클라이언트가 파일에 피드백을 남길 수 있습니다. 파일 피드백은 태스크 또는 주문 내에서만 사용 가능하며, 서비스 카탈로그나 프로필 이미지에서는 사용할 수 없습니다.
</Note>

## 클라이언트 요청 태스크

클라이언트는 포털에서 태스크 요청을 제출할 수 있어, 승인 프로세스를 우회하지 않고 추가적인 필요 사항을 체계적으로 전달할 수 있습니다.

### 클라이언트 워크플로우

<Steps>
  <Step title="요청 접근">
    클라이언트는 클라이언트 포털 사이드바에서 **Requests**를 클릭하거나, 주문 내에서 **+ Request a Task** 버튼을 직접 사용합니다.
  </Step>

  <Step title="요청 양식 작성">
    클라이언트는 다음을 입력합니다:
    - **Order** (필수 — 요청은 기존 주문에 연결되어야 함)
    - **Task Name** (필수)
    - **Description** (선택 사항)
    - **Attachments** (선택 사항 — 최대 1GB; PDF, DOCX, JPG, PNG, MP4, ZIP 등 다양한 형식 지원)
    - **Start and End Dates** (선택 사항)
  </Step>

  <Step title="요청 제출">
    클라이언트가 **Request**를 클릭합니다. 태스크는 포털에 **Pending**으로 나타나고 에이전시는 알림을 받습니다.
  </Step>
</Steps>

### 에이전시 워크플로우

<Steps>
  <Step title="수신 요청 보기">
    에이전시 포털 사이드바에서 **Requests**를 클릭합니다. 모든 대기 중인 요청이 목록 보기로 나타납니다. 클라이언트 또는 주문 ID로 필터링하거나, 알파벳순 또는 발행 날짜순으로 정렬하고, **Preferences**를 사용하여 열을 표시하거나 숨길 수 있습니다.
  </Step>

  <Step title="요청 검토">
    요청을 클릭하여 결정을 내리기 전에 설명을 읽고 첨부 파일을 미리 봅니다.
  </Step>

  <Step title="승인 또는 거부">
    각 요청 옆의 작업 버튼을 사용하여 개별적으로 승인 또는 거부하거나, 오른쪽 상단의 일괄 작업을 사용하여 모든 요청을 한 번에 승인 또는 거부합니다.

    - **승인된** 요청은 주문의 활성 태스크가 되어 일반 태스크 워크플로우를 따릅니다.
    - **거부된** 요청은 **Rejected** 상태로 기록되며 주문에 추가되지 않습니다.
  </Step>
</Steps>

<Note>
클라이언트는 승인 단계를 우회할 수 없습니다. 요청은 라이브 태스크가 되기 전에 항상 에이전시의 조치가 필요합니다. 대기 중인 요청은 결정이 내려질 때까지 클라이언트와 에이전시 모두에게 표시됩니다.
</Note>

## 전체 태스크 보기

개별 주문 내에서 태스크를 관리하는 것 외에도, 단일 화면에서 모든 주문의 태스크를 볼 수 있습니다. 왼쪽 사이드바에서 **Tasks**로 이동하여 **All Tasks** 페이지에 접근합니다. 여기서 다음을 수행할 수 있습니다:

- 어떤 주문에 속하는지에 관계없이 가시성이 있는 모든 태스크 보기
- 상태, 담당자, 마감 일자, 또는 주문으로 필터링 및 정렬
- 전체 작업량에서 기한이 지났거나 위험에 처한 항목 빠르게 파악

이러한 주문 전체 가시성은 에이전시 수준에서 팀 역량을 균형 있게 맞추고 납품 상태를 추적해야 하는 프로젝트 매니저와 어드민에게 특히 유용합니다.
