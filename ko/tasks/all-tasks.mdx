AgencyHandy의 전체 태스크 페이지는 에이전시 전반의 활성 워크플로우에 속한 모든 태스크를 단일 중앙 집중식 보기로 제공합니다. 개별 주문을 일일이 찾아다닐 필요 없이 한 곳에서 모든 관련 태스크를 보고, 담당자, 우선순위, 날짜별로 필터링하고, 목록에서 직접 수정할 수 있습니다. 가시성은 역할에 따라 자동으로 범위가 지정됩니다 — 슈퍼 어드민은 모든 것을 볼 수 있고, 매니저는 배정된 주문만, 직원은 본인 태스크만 볼 수 있습니다.

## 전체 태스크 페이지 접근

1. AgencyHandy 계정에 로그인합니다.
2. 왼쪽 사이드바에서 **Tasks**를 클릭합니다.
3. **All Tasks**를 선택하여 중앙 집중식 태스크 대시보드를 엽니다.

<Note>
  전체 태스크 페이지에는 **Ongoing** 또는 **In Review** 상태인 주문의 태스크만 표시됩니다. 다른 상태(예: Completed, Pending)의 주문에 연결된 태스크는 여기에 표시되지 않습니다.
</Note>

## 역할별 태스크 가시성

전체 태스크 페이지에서 보이는 내용은 전적으로 역할에 따라 달라집니다.

| 역할 | 볼 수 있는 내용 |
|------|---------------|
| **슈퍼 어드민 / 어드민** | 모든 주문, 팀, 상태에 걸친 모든 태스크 |
| **프로젝트 매니저 (PM)** | 매니저로 배정된 주문의 태스크만 |
| **직원 / 배정자** | 직접 배정된 태스크만 — 같은 주문에서 동료가 작업 중이더라도 |

<Tip>
  PM이 볼 것으로 예상되는 태스크를 찾지 못하는 경우, 해당 주문에서 PM이 매니저로 배정되어 있는지 확인하세요. PM은 해당 주문의 배정된 매니저가 아니면 태스크를 볼 수 없습니다.
</Tip>

## 태스크가 이 페이지에 나타나는 시점

다음 조건이 충족될 때 태스크가 전체 태스크 페이지에 표시됩니다:

<CardGroup cols={2}>
  <Card title="태스크 가시성을 트리거하는 주문" icon="circle-check">
    - 수동 주문(일회성 또는 구독)이 생성됨
    - Stripe 또는 수동 결제 방법을 통한 카탈로그 주문이 접수됨
    - 에이전시에 의해 제안서가 수락됨
  </Card>
  <Card title="태스크를 숨기는 주문" icon="eye-slash">
    - 에이전시가 아직 수락하지 않은 클라이언트 요청 태스크
    - 에이전시가 아직 수락하지 않은 트라이얼 주문
    - 수동 은행 결제 확인을 기다리는 대기 중 상태의 주문
  </Card>
</CardGroup>

## 검색, 필터링, 정렬

전체 태스크 페이지에는 원하는 것을 정확하게 찾을 수 있도록 도와주는 완전한 제어 세트가 포함되어 있습니다.

### 검색

검색 바를 사용하여 이름, 담당자, 또는 관련 주문으로 태스크를 찾습니다. 입력하는 즉시 결과가 실시간으로 업데이트됩니다.

### 필터

**Filter → + Add Filter**를 클릭하여 필터 패널을 엽니다. 정밀한 결과를 위해 여러 필터를 조합할 수 있습니다:

| 필터 | 기능 |
|--------|--------------|
| **Order ID** | 특정 주문 번호에 연결된 태스크 표시 |
| **Assignee** | 특정 팀 구성원에게 배정된 태스크 표시 |
| **Status** | 태스크 진행 상태별 필터링: Pending, In Progress, Review, Done 등 |
| **Priority** | 높음, 중간, 낮음 우선순위 태스크에 집중 |
| **Label** | 태스크 레이블 또는 카테고리로 범위 좁히기 |
| **Kick Off Date** | 특정 날짜 이후에 시작되는 태스크 표시 |
| **End Date** | 특정 날짜까지 마감되는 태스크 표시 |

<Note>
  필터는 전체 태스크 페이지에서만 사용할 수 있습니다. 브라우저 세션 동안 유지되지만 페이지 새로 고침 시 초기화됩니다. 필터는 본인의 보기에만 영향을 미치며 다른 사용자가 보는 것은 변경하지 않습니다.
</Note>

모든 필터를 지우려면 **Clear Filters**를 클릭하여 전체 필터링되지 않은 목록으로 돌아갑니다.

### 정렬

다음 기준으로 태스크 목록을 정렬합니다:
- **마감 날짜** — 다가오거나 이미 지난 태스크 표시
- **우선순위** — 가장 중요한 태스크를 상단으로
- **생성 시간** — 최근에 추가된 태스크를 먼저 보기

### 이동 경로 탐색

페이지 상단의 이동 경로를 사용하여 하위 태스크, 태스크 목록, 상위 주문 간 탐색 시 맥락을 유지합니다. 특정 태스크를 자세히 살펴본 후 더 넓은 목록으로 돌아가야 할 때 특히 유용합니다.

## 태스크 수정

전체 태스크 페이지에서 두 가지 방법으로 태스크를 수정할 수 있습니다:

<Tabs>
  <Tab title="인라인 수정">
    태스크 행 내에서 수정 가능한 필드(상태, 담당자, 마감 날짜 등)를 직접 클릭하여 목록 보기를 벗어나지 않고 변경합니다. 빠른 업데이트에 가장 빠른 옵션입니다.
  </Tab>
  <Tab title="모달 수정">
    태스크 이름을 클릭하여 전체 태스크 상세 모달을 엽니다. 여기서 모든 태스크 속성을 업데이트하고, 메모를 추가하고, 하위 태스크를 보고, 완전한 태스크 이력을 확인할 수 있습니다. 더 철저한 수정이 필요하거나 맥락이 필요할 때 사용합니다.
  </Tab>
</Tabs>

## 태스크 삭제 규칙

전체 태스크 페이지의 태스크는 상위 주문에 직접 연결되어 있습니다.

- **주문이 삭제되면** 해당 주문과 연결된 모든 태스크가 전체 태스크 페이지에서 **자동으로 영구 제거**됩니다.
- 이 삭제는 모든 사용자 역할에 영향을 미칩니다 — 주문이 삭제된 후에는 누구도 해당 태스크를 볼 수 없습니다.
- 주문 삭제로 삭제된 태스크에 대한 **복구 옵션은 없습니다**.

<Warning>
  주문을 삭제하면 모든 태스크가 영구적으로 제거됩니다. 주문을 삭제하기 전에 중요한 태스크 이력이나 납품물 기록을 다른 곳에 저장해 두세요.
</Warning>

## 추가 참고 사항

<Accordion title="실시간 업데이트">
  주문, 태스크 배정, 또는 태스크 상태 변경 사항은 전체 태스크 페이지에 실시간으로 반영됩니다. 페이지를 열어 두는 동안 동료가 태스크를 업데이트하면 새로 고침 없이 변경 사항을 볼 수 있습니다.
</Accordion>

<Accordion title="클라이언트 요청 태스크">
  클라이언트가 제출한 태스크는 에이전시가 명시적으로 수락할 때까지 전체 태스크 페이지에 나타나지 않습니다. 수락될 때까지 이러한 태스크는 대기 상태로 남아 있으며 중앙 집중식 보기에서는 보이지 않습니다.
</Accordion>

<Accordion title="태스크와 주문 상태">
  태스크는 Ongoing 또는 In Review 상태의 주문에서만 나타납니다. 주문이 Completed로 이동하거나 일시 중지되면 해당 태스크는 더 이상 여기에 표시되지 않습니다. 이러한 상태 외의 주문에 연결된 태스크를 보려면 해당 주문으로 직접 이동하세요.
</Accordion>
