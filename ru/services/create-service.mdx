Создание услуги в AgencyHandy — это первый шаг к продаже предложений вашего агентства через платформу. Одна услуга может содержать до пяти ценовых тарифных планов, автоматизированные задачи по умолчанию, которые запускаются при размещении заказа, FAQ для снижения вопросов перед покупкой, прикреплённое портфолио для социального доказательства и необязательную форму опроса — всё настраивается в одном месте перед публикацией.

<Note>
  Для создания услуг необходима роль **Суперадминистратора** или **Администратора**. Другие роли требуют явных разрешений на услуги для доступа к этому разделу.
</Note>

<Steps>
  <Step title="Выберите тип услуги">
    Откройте **Услуги** в левой боковой панели и нажмите **+ Услуга**, чтобы начать новое предложение.

    В верхней части страницы выберите модель оплаты:

    - **Разовая услуга** — клиент платит единовременную сумму и получает результат один раз. Лучше всего подходит для проектов с чётким объёмом: создание веб-сайтов, пакеты логотипов, аудиты и т. д.
    - **Услуга по подписке** — оплата повторяется автоматически по заданному вами расписанию. Лучше всего подходит для абонементного обслуживания, постоянного создания контента, управляемых услуг или планов технического обслуживания.

    <Warning>
      Тип услуги фиксируется после сохранения. Если вам нужна другая модель оплаты, создайте новую услугу.
    </Warning>
  </Step>

  <Step title="Добавьте сведения об услуге">
    Разверните раздел **Сведения об услуге** и заполните следующие поля:

    **Название услуги** *(обязательно)*
    Так услуга отображается для клиентов везде на платформе — в каталоге, в счетах и в рабочем пространстве заказа. Сделайте название чётким и описательным.

    **Загрузка медиафайлов**
    Загрузите одно или несколько изображений для визуального представления услуги. Высококачественные изображения повышают доверие клиентов и коэффициент конверсии.

    **Описание услуги**
    Напишите подробное объяснение того, что включает услуга, для кого она предназначена и чего могут ожидать клиенты. Используйте это пространство для дифференциации вашего предложения.

    **Настройки налога**
    Выберите, является ли налог **включённым** (уже включён в цену) или **исключённым** (добавляется сверху при оформлении заказа).

    **Валюта**
    Установите валюту для всех тарифных планов в этой услуге. Все планы в рамках одной услуги используют одну и ту же валюту.

    <Tip>
      Загружайте изображения для предварительного просмотра в соотношении 16:9 для наилучшего отображения в клиентском портале.
    </Tip>
  </Step>

  <Step title="Создайте тарифные планы">
    Разверните **Сведения о тарифных планах** и нажмите **+ Новый тарифный план**, чтобы добавить первый ценовой уровень. Вы можете создать до **5 тарифных планов** на услугу.

    Для каждого тарифного плана настройте следующее:

    **Название тарифного плана** *(обязательно)*
    Название, которое видят клиенты, например *Базовый*, *Стандартный* или *Премиум*.

    **Значок тарифного плана** *(необязательно)*
    Выберите встроенный значок или загрузите пользовательский. Загружаемые значки должны быть размером **32 × 32 пикселя**.

    **Медиафайл тарифного плана** *(необязательно)*
    Загрузите обложку, специфичную для этого тарифного плана. Поддерживаемые форматы: PNG, JPG, JPEG. Максимальный размер файла: **5 МБ**. Максимум 1 файл на тарифный план. Рекомендуемое соотношение: **16:9**.

    **Краткое описание тарифного плана** *(необязательно)*
    Краткое резюме в одну строку с описанием результатов этого тарифного плана. Максимум **140 символов**.

    **Что включено** *(обязательно)*
    Перечислите конкретные результаты, включённые в этот тарифный план. Вы можете добавить до **15 позиций** на тарифный план. Каждый пункт должен описывать один конкретный результат (например, *«5 записей в блоге в месяц, до 1000 слов каждая»*).

    <Tip>
      После создания нескольких тарифных планов перетащите их для изменения порядка отображения для клиентов. Отметьте один как **Самый популярный**, чтобы привлечь внимание к рекомендуемому уровню.
    </Tip>
  </Step>

  <Step title="Установите цены и правила доставки">
    Внутри каждого тарифного плана установите основные ценовые поля и необязательные параметры управления:

    **Сумма** *(обязательно)*
    Цена, которую клиент платит за этот тарифный план.

    **Срок выполнения** *(обязательно)*
    Предполагаемое количество дней для предоставления услуги после покупки.

    **Необязательные параметры управления ценообразованием:**

    <Accordion title="Добавить скидку">
      Примените процентное или фиксированное снижение цены для этого тарифного плана. Цена со скидкой отображается клиентам рядом с исходной ценой, создавая эффективный рекламный эффект.
    </Accordion>

    <Accordion title="Добавить пробный период (только для подписки)">
      Предложите бесплатный или льготный пробный период до начала регулярной оплаты подписки. Этот вариант доступен только для услуг типа «Подписка».
    </Accordion>

    <Accordion title="Добавить ограничение количества">
      Ограничьте количество единиц, которые клиент может приобрести за один сеанс оформления заказа. Используйте это для тарифных планов с ограниченной доступностью или услуг с ограниченной пропускной способностью.
    </Accordion>

    <Accordion title="Добавить дополнительные сведения о цене">
      Разбейте общую стоимость на отдельные позиции — например, показывая *базовую цену + сбор за подключение + стоимость единицы*. Каждая метка разбивки поддерживает максимум **20 символов**.
    </Accordion>

    <Note>
      Все обязательные поля (Сумма и Срок выполнения) должны быть заполнены для каждого тарифного плана, прежде чем вы сможете опубликовать услугу.
    </Note>
  </Step>

  <Step title="Добавьте задачи по умолчанию для автоматизации">
    Каждый тарифный план включает раздел **Задачи по умолчанию**. Задачи, определённые здесь, автоматически создаются в рабочем пространстве заказа в момент покупки клиентом этого тарифного плана — ручная настройка после каждой продажи не требуется.

    Нажмите **+ Добавить ещё** для открытия модального окна создания задачи. Для каждой задачи можно настроить:

    | Поле | Подробности |
    |---|---|
    | **Название задачи** | Обязательно; максимум **160 символов** |
    | **Описание** | Подробные инструкции для исполнителя |
    | **Исполнитель** | Член команды, ответственный за выполнение задачи |
    | **Вложение** | Загрузите справочные файлы; поддерживаемые форматы: csv, odt, doc, docx, htm, html, pdf, ppt, pptx, txt, xls, xlsx, jpg, jpeg, png, gif, webp, svg, ai, mp4, mp3, wav, zip, rar; макс. размер файла **1 ГБ** на файл |
    | **Метка** | Пометьте задачу для фильтрации и отчётности |
    | **Статус** | Начальный статус при создании задачи |
    | **Проверка клиентом** | Включите, если эта задача требует одобрения клиента перед завершением |

    Добавьте столько задач, сколько необходимо для полного охвата процесса доставки для этого тарифного плана. Последовательные задачи по умолчанию гарантируют, что каждый заказ выполняется по одному и тому же контролируемому рабочему процессу.

    <Tip>
      Сопоставьте задачи по умолчанию со своими внутренними стандартными операционными процедурами. Например, тарифный план *Управление социальными сетями* может автоматически создавать задачи *Черновик контент-плана*, *Одобрение клиента*, *Публикация постов* и *Ежемесячный отчёт*.
    </Tip>
  </Step>

  <Step title="Добавьте FAQ и портфолио">
    Разверните раздел **Дополнительная информация**, чтобы добавить два необязательных, но важных элемента:

    **Прикрепление портфолио**
    Свяжите категорию портфолио с этой услугой, чтобы клиенты могли просматривать соответствующие образцы вашей прошлой работы прямо в карточке услуги. Вы можете подключить существующую категорию портфолио или создать новую прямо на этом экране.

    **Часто задаваемые вопросы (FAQ)**
    Нажмите **+ Добавить**, чтобы вставить пару «вопрос-ответ». Добавьте столько FAQ, сколько необходимо — распространённые темы включают:

    - Оригинальность контента и права
    - Процесс правок и обратной связи
    - Границы объёма и что *не* включено
    - Сроки и частота коммуникаций
    - Политика оплаты и возврата

    Хорошо написанные FAQ снижают количество вопросов до покупки и помогают клиентам самостоятельно оценить соответствие.
  </Step>

  <Step title="Прикрепите форму опроса">
    Форма опроса собирает информацию, необходимую от клиента до начала работы — например, фирменные руководства, учётные данные для доступа, целевую аудиторию или цели проекта.

    - Нажмите **+ Создать новую форму опроса**, чтобы создать форму с нуля, или выберите существующую форму из выпадающего списка.
    - Если для этой услуги не требуется форма опроса, нажмите **Пропустить**, чтобы обойти этот раздел.
    - Нажмите **Обновить**, чтобы сохранить выбор формы опроса и продолжить.

    <Note>
      Форма опроса необязательна, но настоятельно рекомендуется для услуг, требующих специфических входных данных от клиента до начала выполнения.
    </Note>
  </Step>

  <Step title="Опубликуйте услугу">
    В нижней части редактора услуги:

    1. Установите переключатель **Опубликовать эту услугу** в положение ВКЛ.
    2. Нажмите **Сохранить** (или **Обновить услугу** при редактировании существующей услуги).

    После публикации услуга немедленно становится видна клиентам на портале. Если оставить переключатель выключенным, услуга сохраняется как черновик и видна только вашей команде.

    <Tip>
      Используйте режим черновика для подготовки услуги заранее и публикации в момент запуска — это удобно для сезонных акций или анонса новых предложений.
    </Tip>
  </Step>
</Steps>

## Настройка дополнений

Дополнения — это необязательные расширения, которые клиенты могут выбрать при оформлении заказа — например, *Срочное выполнение*, *Дополнительные правки* или *Ежемесячный отчёт*. Они позволяют проводить дополнительные продажи сопутствующих услуг без создания совершенно новых предложений.

### Создание дополнения

<Steps>
  <Step title="Откройте раздел «Дополнения»">
    Перейдите в **Услуги** на боковой панели, нажмите кнопку **стрелка вниз** рядом с заголовком «Услуги» и выберите **Дополнения**.
  </Step>
  <Step title="Создайте новое дополнение">
    Нажмите **+ Добавить новое дополнение** и заполните:

    - **Название дополнения**
    - **Описание** — используйте редактор с форматированием для чёткого объяснения того, что включает дополнение
    - **Тип ценообразования** — выберите *Разовый* или *Подписка*
    - **Сведения о цене:**
      - *Разовый:* введите сумму и валюту
      - *Подписка:* установите интервал оплаты (например, каждый 1 месяц), необязательный лимит циклов оплаты, сумму подписки и валюту
  </Step>
  <Step title="Опубликуйте и сохраните">
    Установите переключатель **Опубликовать** в положение ВКЛ, затем нажмите **Создать**. Только опубликованные дополнения видны при их подключении к формам заказов.
  </Step>
</Steps>

### Прикрепление дополнения к форме заказа

<Steps>
  <Step title="Откройте форму заказа">
    Перейдите в **Формы** и откройте нужную форму заказа.
  </Step>
  <Step title="Выберите услугу">
    Выберите услугу, к которой хотите прикрепить дополнение.
  </Step>
  <Step title="Добавьте дополнения">
    Выберите раздел **Дополнения** на левой панели, затем выберите дополнения, которые хотите связать с этой услугой.
  </Step>
  <Step title="Сохраните форму">
    Нажмите **Сохранить**. Теперь клиенты будут видеть выбранные дополнения как необязательные расширения при оформлении заказа.
  </Step>
</Steps>

<Note>
  Дополнения должны использовать ту же валюту, что и услуга, к которой они прикреплены. Если валюты не совпадают, дополнение не появится в списке выбора.
</Note>

### Поведение оплаты при подписке на дополнения

Когда циклы оплаты услуги и дополнения по подписке различаются, AgencyHandy автоматически управляет пересечением:

<Accordion title="У дополнения больше циклов, чем у услуги">
  Услуга и дополнение оплачиваются вместе в течение срока действия услуги. После завершения услуги дополнение продолжает оплачиваться отдельно в течение оставшихся циклов.

  **Пример:** Услуга работает 4 цикла, дополнение — 5 циклов → оплачиваются вместе 4 цикла, дополнение оплачивается отдельно ещё 1 цикл.
</Accordion>

<Accordion title="У дополнения меньше циклов, чем у услуги">
  Услуга и дополнение оплачиваются вместе в течение срока действия дополнения. После завершения дополнения услуга продолжает оплачиваться отдельно в течение оставшихся циклов.

  **Пример:** Услуга работает 5 циклов, дополнение — 3 цикла → оплачиваются вместе 3 цикла, услуга оплачивается отдельно ещё 2 цикла.
</Accordion>

## Редактирование услуги

После публикации услуги вы можете обновить любое поле, не прерывая существующие заказы.

1. Перейдите в **Услуги** и найдите услугу, которую хотите обновить.
2. Нажмите **меню из трёх точек** (⋮) с правой стороны строки услуги.
3. Выберите **Редактировать**. Откроется редактор услуг со всеми существующими настройками.
4. Внесите изменения — обновите название, описание, цены, медиафайлы, тарифные планы, задачи или FAQ.
5. Нажмите **Обновить услугу** для сохранения.

<Warning>
  Изменения цен тарифных планов или сроков выполнения вступают в силу **только для новых заказов**. Существующие заказы не затрагиваются ретроспективно.
</Warning>

## Справочник по полям

| Поле | Ограничение |
|---|---|
| Тарифные планы на услугу | До 5 |
| Результаты на тарифный план (Что включено) | До 15 |
| Краткое описание тарифного плана | 140 символов |
| Размер значка тарифного плана | 32 × 32 пикселя |
| Размер медиафайла тарифного плана | 5 МБ (PNG, JPG, JPEG) |
| Название задачи | 160 символов |
| Размер вложения задачи | 1 ГБ на файл |
| Метка разбивки цены | 20 символов |
