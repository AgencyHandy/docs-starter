Страница «Все задачи» в AgencyHandy предоставляет единое централизованное представление каждой задачи, относящейся к активным рабочим процессам вашего агентства. Вместо того чтобы искать задачи в отдельных заказах, вы можете видеть все нужные задачи в одном месте, фильтровать их по исполнителю, приоритету или дате и вносить правки прямо из списка. Видимость автоматически ограничивается вашей ролью — Супер-администраторы видят всё, Менеджеры проекта видят свои назначенные заказы, а Сотрудники видят только свои задачи.

## Доступ к странице «Все задачи»

1. Войдите в свою учётную запись AgencyHandy.
2. Нажмите **Tasks** на левой боковой панели.
3. Выберите **All Tasks**, чтобы открыть централизованный дашборд задач.

<Note>
  Страница «Все задачи» отображает только задачи из заказов в статусах **Ongoing** или **In Review**. Задачи, привязанные к заказам в любом другом состоянии (например, Completed, Pending), здесь не отображаются.
</Note>

## Видимость задач по ролям

То, что вы видите на странице «Все задачи», полностью зависит от вашей роли.

| Роль | Что они видят |
|------|---------------|
| **Супер-администратор / Администратор** | Все задачи по каждому заказу, команде и статусу |
| **Менеджер проекта (PM)** | Только задачи из заказов, где он назначен менеджером |
| **Сотрудник / Исполнитель** | Только задачи, непосредственно назначенные ему — даже если коллеги работают в том же заказе |

<Tip>
  Если менеджер проекта не может найти ожидаемую задачу, убедитесь, что он назначен менеджером в соответствующем заказе. Менеджеры проекта не видят задачи, если они не являются назначенным менеджером для данного заказа.
</Tip>

## Когда задачи появляются на этой странице

Задачи отображаются на странице «Все задачи» при выполнении следующих условий:

<CardGroup cols={2}>
  <Card title="Заказы, открывающие видимость задач" icon="circle-check">
    - Создаются ручные заказы (единовременные или по подписке)
    - Размещаются каталожные заказы через Stripe или ручные методы оплаты
    - Коммерческое предложение принято агентством
  </Card>
  <Card title="Заказы, при которых задачи скрыты" icon="eye-slash">
    - Задачи, запрошенные клиентом, которые агентство ещё не приняло
    - Пробные заказы, которые агентство ещё не приняло
    - Заказы в статусе ожидания с ожиданием подтверждения ручного банковского платежа
  </Card>
</CardGroup>

## Поиск, фильтрация и сортировка

Страница «Все задачи» включает полный набор элементов управления для поиска именно того, что вам нужно.

### Поиск

Используйте строку поиска для нахождения задач по названию, исполнителю или связанному заказу. Результаты обновляются в реальном времени по мере ввода.

### Фильтры

Нажмите **Filter → + Add Filter**, чтобы открыть панель фильтров. Вы можете комбинировать несколько фильтров для получения точных результатов:

| Фильтр | Что он делает |
|--------|--------------|
| **Order ID** | Показывает задачи, связанные с конкретным номером заказа |
| **Assignee** | Показывает задачи, назначенные конкретному участнику команды |
| **Status** | Фильтрует по прогрессу задачи: Pending, In Progress, Review, Done и др. |
| **Priority** | Фокусирует на задачах высокого, среднего или низкого приоритета |
| **Label** | Сужает выборку по меткам или категориям задач |
| **Kick Off Date** | Показывает задачи, начинающиеся с определённой даты или после неё |
| **End Date** | Показывает задачи с дедлайном не позднее указанной даты |

<Note>
  Фильтры доступны только на странице «Все задачи». Они сохраняются в течение сессии браузера, но сбрасываются при обновлении страницы. Фильтры влияют только на ваше представление и не изменяют то, что видят другие пользователи.
</Note>

Чтобы очистить все фильтры, нажмите **Clear Filters** для возврата к полному нефильтрованному списку.

### Сортировка

Сортируйте список задач по:
- **Сроку выполнения** — выводите на первый план задачи с приближающимися или уже просроченными сроками
- **Приоритету** — поднимайте наиболее критичные задачи вверх
- **Времени создания** — просматривайте недавно добавленные задачи первыми

### Навигация по хлебным крошкам

Используйте навигационную цепочку в верхней части страницы для сохранения контекста при переходе между подзадачами, списками задач и родительскими заказами. Это особенно удобно при детальном просмотре конкретной задачи и последующем возврате к общему списку.

## Редактирование задач

Вы можете изменять задачи на странице «Все задачи» двумя способами:

<Tabs>
  <Tab title="Встроенное редактирование">
    Нажмите прямо на редактируемое поле в строке задачи — например, статус, исполнителя или срок выполнения — и внесите изменения, не покидая представление списка. Это самый быстрый вариант для быстрых обновлений.
  </Tab>
  <Tab title="Редактирование в модальном окне">
    Нажмите на название задачи, чтобы открыть полное модальное окно деталей задачи. Отсюда вы можете обновить все свойства задачи, добавить заметки, просмотреть подзадачи и увидеть полную историю задачи. Используйте этот вариант для более тщательного редактирования или когда вам нужен контекст.
  </Tab>
</Tabs>

## Правила удаления задач

Задачи на странице «Все задачи» напрямую привязаны к родительским заказам.

- При **удалении заказа** все задачи, связанные с этим заказом, **автоматически и безвозвратно удаляются** со страницы «Все задачи».
- Это удаление затрагивает все роли пользователей — никто не сможет увидеть эти задачи после удаления заказа.
- **Возможность восстановления задач**, удалённых вместе с заказом, **отсутствует**.

<Warning>
  Удаление заказа безвозвратно удаляет все его задачи. Убедитесь, что вся важная история задач или записи о результатах работы сохранены в другом месте перед удалением заказа.
</Warning>

## Дополнительные замечания

<Accordion title="Обновления в реальном времени">
  Изменения в заказах, назначениях задач или статусах задач отражаются на странице «Все задачи» в реальном времени. Если коллега обновит задачу, пока у вас открыта страница, вы увидите изменение без необходимости обновлять страницу.
</Accordion>

<Accordion title="Задачи по запросу клиента">
  Задачи, отправленные клиентами, не появляются на странице «Все задачи» до тех пор, пока агентство явно не примет их. До принятия эти задачи остаются в состоянии ожидания и не видны в централизованном представлении.
</Accordion>

<Accordion title="Задачи и статус заказа">
  Задачи отображаются только для заказов в статусе Ongoing или In Review. Если заказ перемещается в статус Completed или приостанавливается, его задачи больше здесь не отображаются. Перейдите к конкретному заказу для просмотра задач, привязанных к заказам с другими статусами.
</Accordion>
