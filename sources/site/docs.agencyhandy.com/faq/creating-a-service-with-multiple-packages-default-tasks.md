**Creating a Service with Multiple Packages, Default Tasks & FAQs in AgencyHandy**

1. **Who can create a service in AgencyHandy?** Only Super Admins and Admins with the necessary permissions can create services in AgencyHandy.
 
2. **How do I access the service creation feature?** Log in to your AgencyHandy account, navigate to the "Services" section from the left column, and click on the "Service" button to open the creation form.
 
3. **What information is required to create a new service?** You need to provide a Service Name, upload a Cover Image, add a Description, set Pricing Options, and define the Duration and Currency.
 
4. **What are the accepted formats and size limits for the cover image?** The cover image should be in PNG, JPG, or JPEG format and must not exceed 5 MB in size.
 
5. **Can I set one-time and recurring payment options for services?** Yes, you can specify a fixed price for one-time payments and offer subscription-based services with recurring payment options.
 
6. **How do I define the duration of a service?** During the service setup, you can specify the duration in days, weeks, months, or years to tailor your services accordingly.
 
7. **Is it possible to create multiple packages for a single service?** Yes, you can create up to 5 packages for each service by clicking the "+ Add More Packages" button.
 
8. **What details are required for each package?** For each package, you need to provide a Package Name, a Brief description, list up to 15 offerings, define the Duration, set the Amount, and optionally add a Trial Period.
 
9. **How do I add a trial period to a package?** Click the "Add Trial Period" button during package setup and specify the number of days for the free trial.
 
10. **Can I add default tasks to a service?** Yes, you can add default tasks by clicking the "Describe Service" button during the service creation process.
 
11. **Are FAQs mandatory when creating a service?** No, adding FAQs is optional, but recommended to help clients understand your service better.
 
12. **How do I add FAQs to my service?** Click the "+ Add Questions" button in the "Describe Service" section to add frequently asked questions.
 
13. **What is an intake form, and is it necessary?** An intake form is used to collect necessary information from clients before starting the service. It's optional but recommended.
 
14. **How can I add an intake form to my service?** Click "+ Create New Intake Form" to create a new form or select an existing one during the service setup.
 
15. **What happens if I skip adding an intake form?** If you skip this section, the service will still be created, but you'll need to collect client information separately.
 
16. **How do I publish my service after setup?** After completing all sections, activate the radio button to publish your service offerings and click the "Update" button.
 
17. **Can I edit a service after it's been published?** Yes, you can access the "Services" section, locate the service, click the three-dot button on the right, and choose the edit button to make changes.
 
18. **Is there a limit to the number of services I can create?** AgencyHandy does not impose a limit on the number of services you can create.
 
19. **Can I delete a package after it's been published?** Yes, you can delete a package at any time from the "Services" section in your dashboard.
 
20. **What should I do if I want to add more than five packages to a service?** Currently, AgencyHandy allows a maximum of five packages per service. For more packages, consider creating a new service.
 
21. **Is it possible to clone a service with its packages and tasks?** At the moment, AgencyHandy does not support direct cloning of services. Each service needs to be created independently.
 
22. **Will my service be visible to clients immediately after creation?** No, you need to click the "Publish" button after setup to make it visible to clients.
 
23. **Can I change the currency type after creating the service?** Yes, you can update the currency type by editing the service details in the "Services" section.
 
24. **Is there an approval process before a service goes live?** No, once you click the "Publish" button, the service becomes live instantly.
 
25. **How do I handle price changes for existing packages?** You can edit the package pricing anytime through the "Services" section in your dashboard.
 
26. **Are clients notified when I update a service?** Clients are automatically notified when a service is updated through notifications.
 
27. **Can I organize multiple packages under one service?** Yes, while setting up your service, click the "+ Add More Packages" button and enter the details for each package to keep them organized.
 
28. **What if I want to offer both one-time and recurring payment options in a single package?** Currently, each package can have either a one-time payment or a recurring payment option, not both.
 
29. **Can I set different durations for each package within a service?** Yes, you can define the duration for each package individually during the setup.
 
30. **Is there a way to preview the service before publishing it?** While there's no dedicated preview mode, you can review all the entered information before clicking the "Publish" button.
 
31. **Can I duplicate an existing package to create a similar one?** Currently, you need to manually enter the details for each package; duplication is not supported.
 
32. **How do I ensure the service name is effective?** Ensure the service name is descriptive and easy to understand for potential clients.
 
33. **What are the benefits of adding FAQs to my service?** Providing comprehensive answers to common questions assists clients in understanding your service better.
 
34. **Can I update the intake form associated with a service after publishing?** Yes, you can edit the intake form by accessing the service details and making the necessary changes.
 
35. **Is there a way to categorize services for better organization?** While the current setup focuses on individual services, consider using clear naming conventions to categorize them effectively.
