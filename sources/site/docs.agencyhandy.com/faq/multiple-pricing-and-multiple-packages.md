FAQ: Multiple Pricing and Multiple Packages in AgencyHandy

#### 

Multiple Pricing Options

1. **What are multiple pricing options in AgencyHandy?** Multiple pricing options allow you to offer various pricing models for your services, including one-time payments and subscription-based options.
 
2. **Can I offer both one-time and subscription-based pricing for the same service?** Yes, AgencyHandy allows you to set both one-time and subscription-based pricing for enhanced flexibility.
 
3. **How many pricing tiers can I create for one-time payments?** Based on your service offerings, you can create multiple pricing tiers, such as Tier 1: $100, Tier 2: $200, and Tier 3: $300.
 
4. **Is it possible to set different billing intervals for subscription-based pricing?** Yes, you can configure weekly, monthly, or yearly billing cycles for subscription services.
 
5. **Do I need admin permissions to configure multiple pricing options?** Yes, administrative permissions are required to configure pricing options in AgencyHandy.
 
6. **Will the invoice automatically reflect the selected pricing model?** Yes, once a pricing option is selected, the invoice will be generated accordingly.
 
7. **Can clients switch between one-time and subscription models after purchase?** No, clients must select the pricing model at the time of purchase; changes require a new order.
 
8. **Are taxes calculated differently for subscription vs. one-time services?** Taxes are applied based on the pricing model configured during service setup.
 
9. **How do I review the multiple pricing options before publishing?** You can preview the pricing configurations before publishing them to ensure accuracy.
 
10. **Is there a way to compare pricing tiers for clients during checkout?** Yes, AgencyHandy allows clients to view and compare different pricing tiers.
 

---

#### 

Multiple Packages

1. **What is the purpose of multiple packages in AgencyHandy?** Multiple packages allow you to offer up to five package options per service, each with unique features and pricing.
 
2. **Can I add more than five packages to a service?** No, the maximum limit is five packages per service.
 
3. **Do I need to create a service first before adding packages?** Yes, services must be defined before you can add packages.
 
4. **Can I offer both one-time and subscription options within packages?** Yes, each package can be configured with its own pricing model, either one-time or subscription.
 
5. **Can I delete a package if clients have already purchased it?** No, packages with active or pending orders cannot be deleted.
 
6. **How do clients view the available packages for a service?** Clients can navigate to the catalogue, view service details, and see all available packages.
 
7. **Can I edit package details after publishing them?** Yes, you can modify the name, description, and pricing of packages at any time.
 
8. **Do package changes affect existing client orders?** No, changes to packages only apply to new orders, not existing ones.
 
9. **Is there a trial period option for packages?** Yes, you can configure a trial period when setting up the package.
 
10. **How can I set up different pricing for each package?** Pricing is defined during package creation, allowing for individual pricing tiers.
 
11. **What happens if I exceed the maximum number of packages?** The system will prevent you from adding more than five packages per service.
 
12. **Can I reorder how packages appear to clients?** Currently, packages are displayed in the order they are added; reordering is not available.
 
13. **Are package purchases reflected in the client's portal?** Yes, all purchased packages are visible in the client's portal.
 
14. **How do I implement webhooks for package updates?** Webhooks can be configured to trigger on package creation, deletion, and updates.
 
15. **Can I hide certain packages from specific clients?** Currently, packages are visible to all clients unless unpublished the service manually.
