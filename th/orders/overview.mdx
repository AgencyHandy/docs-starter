Orders คือแกนหลักการดำเนินงานของ AgencyHandy Order แสดงถึงคำขอของลูกค้าสำหรับบริการหรือผลิตภัณฑ์ และทำหน้าที่เป็น workspace กลางที่ทีมของคุณติดตามความคืบหน้า, จัดการงาน, แลกเปลี่ยนข้อเสนอแนะ และส่งมอบผลลัพธ์ ไม่ว่าลูกค้าจะสั่ง order ผ่านแคตาล็อกบริการที่เผยแพร่ของคุณหรือทีมของคุณสร้างขึ้นด้วยตนเอง ทุก order จะปฏิบัติตามวงจรชีวิตที่มีโครงสร้างเดียวกันที่ช่วยให้เอเจนซีของคุณเป็นระเบียบและแจ้งให้ลูกค้าทราบ

## สิ่งที่ Order ประกอบด้วย

ทุก order แสดงข้อมูลสำคัญในทันทีและให้สมาชิกทีมมีสถานที่เดียวในการประสานงาน:

| ฟิลด์ | คำอธิบาย |
|---|---|
| **Order ID** | ตัวระบุที่ไม่ซ้ำกันที่สร้างอัตโนมัติสำหรับแต่ละ order |
| **Client** | ลูกค้าที่ order นี้เป็นของ |
| **Service** | บริการที่กำลังส่งมอบ |
| **Project Manager (PM)** | สมาชิกทีมที่รับผิดชอบการส่งมอบ (ไม่บังคับ) |
| **Budget** | ราคาที่ตกลงสำหรับ order |
| **Kick-off Date** | เมื่อไหร่ที่งานเริ่มต้น |
| **Due Date** | กำหนดเส้นตาย order ให้เสร็จสิ้น |
| **Order Brief** | คำแนะนำ, บันทึก และไฟล์อ้างอิงที่แนบของลูกค้า |
| **Tasks** | รายการงานที่แยกย่อยที่มอบหมายให้สมาชิกทีม |
| **Discussions** | การสนทนาแบบ thread ระหว่างพนักงานเอเจนซีและลูกค้า |

## สถานะ Order

ทุก order เคลื่อนผ่านชุดสถานะที่กำหนดไว้ คุณไม่สามารถเพิ่มหรือเปลี่ยนชื่อสถานะเหล่านี้ได้

<Note>
เฉพาะ Super Admins, Admins และ Project Managers เท่านั้นที่สามารถเปลี่ยนสถานะของ order ได้
</Note>

| สถานะ | ความหมาย |
|---|---|
| **Pending** | Order ถูกสร้างขึ้นและรอให้งานเริ่มต้น |
| **Ongoing** | งานกำลังดำเนินอยู่ |
| **In Review** | ผลงานที่ส่งมอบถูกส่งและรอการตรวจสอบจากลูกค้าหรือภายใน |
| **Complete** | Order ถูกส่งมอบอย่างสมบูรณ์และได้รับการยอมรับแล้ว |
| **Cancelled** | Order ถูกยกเลิกและไม่ใช้งานอีกต่อไป |

## การเข้าถึงตามบทบาท

บทบาทต่างๆ ใน AgencyHandy มีความสามารถที่แตกต่างกันภายใน orders:

| บทบาท | สร้าง Orders ได้ | แก้ไข Orders ได้ | ลบ Orders ได้ | เปลี่ยนสถานะได้ | เพิ่มงานได้ | เพิ่มสมาชิกทีมได้ |
|---|---|---|---|---|---|---|
| **Super Admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Admin** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Project Manager (PM)** | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ |
| **Employee** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **Client** | ❌ | ❌ | ❌ | ❌ | ✅ (คำขอเท่านั้น) | ❌ |

<Tip>
ลูกค้าสามารถส่งข้อเสนอแนะและมีส่วนร่วมในการอภิปรายใน orders พนักงานที่ได้รับมอบหมายใน order สามารถดูและทำงานในงานได้แต่ไม่สามารถจัดการ order เองได้
</Tip>

## สิ่งที่คุณสามารถทำกับ Orders

AgencyHandy ให้ชุดเครื่องมือที่ครบวงจรสำหรับการจัดการ orders ตลอดวงจรชีวิต:

- **สร้าง orders** สำหรับลูกค้า — ทั้งจากบริการที่เผยแพร่หรือด้วยตนเองสำหรับบริการที่ยังอยู่ในการพัฒนา
- **ยอมรับและปฏิเสธ** orders ที่ลูกค้าส่งมา พร้อมตัวเลือกในการให้เหตุผลเมื่อปฏิเสธ
- **ดูรายละเอียด order** รวมถึงความคืบหน้า, สมาชิกทีมที่ได้รับมอบหมาย, วันที่เริ่มต้นและกำหนดส่ง, งบประมาณ และ brief
- **เรียงลำดับ, กรอง และค้นหา** ใน orders ทั้งหมดตาม ID, สถานะ, ชื่อลูกค้า หรือเกณฑ์อื่นๆ
- **เพิ่มงาน** โดยตรงภายใน order เพื่อติดตามและมอบหมายหน่วยงานที่แยกย่อย
- **เริ่มการอภิปราย** เพื่อสื่อสารกับลูกค้าและสมาชิกทีมในบริบท

## Orders ด้วยตนเองสำหรับบริการที่ไม่ได้เผยแพร่

คุณไม่ถูกจำกัดให้สร้าง orders เฉพาะสำหรับบริการที่คุณเผยแพร่สาธารณะ Super Admins และ Admins สามารถสร้าง manual orders สำหรับบริการที่ไม่ได้เผยแพร่ — มีประโยชน์เมื่อบริการยังอยู่ระหว่างการพัฒนา, ไม่พร้อมให้บริการชั่วคราว หรือกำลังเสนอให้กับลูกค้าเฉพาะก่อนการเปิดตัวอย่างเป็นทางการ

เมื่อคุณเปิดแบบฟอร์มสร้าง order ดรอปดาวน์ **Service** จะแสดงทั้งบริการที่เผยแพร่และไม่ได้เผยแพร่ ดังนั้นคุณสามารถเลือกอันที่เหมาะสม ฟิลด์ order อื่นๆ และกระบวนการทำงานทั้งหมดจะทำงานเหมือนกับ order มาตรฐาน

## Add-Ons ใน Manual Orders

เมื่อสร้าง manual order คุณสามารถรวม **add-on services** เพื่อให้ลูกค้าได้รับแพ็กเกจที่ปรับแต่งได้นอกเหนือจากบริการหลัก Add-on ต้องถูกสร้างล่วงหน้าและเชื่อมโยงกับบริการที่เกี่ยวข้องในส่วน **Forms** ก่อนที่จะปรากฏเป็นตัวเลือกในแบบฟอร์ม order

<Note>
การเลือก add-on จะปรากฏในแบบฟอร์ม manual order เฉพาะเมื่อมี add-on อย่างน้อยหนึ่งตัวที่เชื่อมต่อกับบริการที่เลือก หากคุณไม่เห็น ให้ยืนยันว่า add-on ของคุณเชื่อมโยงอย่างถูกต้องในการตั้งค่าบริการ
</Note>

เมื่อเลือกบริการที่มี add-on เชื่อมโยงแล้ว คุณสามารถรวม add-on เหล่านั้นก่อนดำเนินการให้เสร็จสิ้นและส่ง order

## การแจ้งเตือน

AgencyHandy จะแจ้งเตือนคนที่ถูกต้องโดยอัตโนมัติเมื่อ orders ดำเนินไปข้างหน้า:

- เมื่อ PM, Admin หรือ Super Admin สร้าง order Super Admins ทั้งหมดในบริษัทของคุณจะได้รับการเข้าถึงและลูกค้าจะได้รับการแจ้งเตือนการสร้าง
- เมื่อสมาชิกทีมถูกเพิ่มเข้าใน order สมาชิกนั้นจะได้รับการแจ้งเตือน
- เมื่อสร้างงานภายใน order ลูกค้าจะได้รับการแจ้งเตือน
