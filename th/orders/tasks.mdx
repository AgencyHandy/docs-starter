งาน (Tasks) คือหน่วยของการทำงานที่เปลี่ยนออเดอร์ให้เป็นผลลัพธ์ที่ส่งมอบได้ แต่ละงานอยู่ภายในออเดอร์ที่เฉพาะเจาะจง สามารถมอบหมายให้สมาชิกทีม และติดตามสถานะ ไทม์ไลน์ เอกสารแนบ และเธรดข้อเสนอแนะของตนเอง หน้านี้อธิบายวิธีสร้างและจัดการงาน วิธีที่ลูกค้าสามารถร้องของาน วิธีที่การติดตามเวลาและการรับฟีดแบ็กไฟล์ทำงาน และวิธีดูงานทั้งหมดข้ามออเดอร์ในที่เดียว

## ข้อกำหนดด้านบทบาท

| การกระทำ | ใครทำได้ |
|---|---|
| สร้างงาน | Super Admin, Admin, Project Manager |
| แก้ไขงาน | Super Admin, Admin, Project Manager |
| ลบงาน | Super Admin, Admin, Project Manager |
| ดูงานที่อยู่ระหว่างตรวจสอบ / เสร็จสิ้น | Super Admin, Admin, PM, ลูกค้า |
| ส่งและตอบฟีดแบ็ก | Super Admin, Admin, PM |
| บันทึกเวลาทำงาน | ผู้ใช้ที่ได้รับมอบหมายทุกคน (Superadmin และ Admin สามารถบันทึกแทนผู้อื่นได้) |
| ร้องของาน (ลูกค้า) | ลูกค้า |
| อนุมัติ / ปฏิเสธงานที่ร้องขอ | Super Admin, Admin |

<Note>
พนักงานที่ได้รับมอบหมายในออเดอร์สามารถเห็นงานที่มีสถานะ **In Review** หรือ **Done** ได้ งานที่อยู่ในสถานะ **In Progress** จะไม่ปรากฏให้พนักงานเห็น
</Note>

## สร้างงาน

<Steps>
  <Step title="เปิดออเดอร์">
    ไปที่ **Orders** ในแถบด้านซ้ายและคลิกออเดอร์ที่ต้องการเพิ่มงาน
  </Step>

  <Step title="เพิ่มงานใหม่">
    ภายในออเดอร์ เลื่อนไปที่ส่วน **Tasks** และคลิก **+ Add New Task** ฟอร์มสร้างงานจะปรากฏขึ้น
  </Step>

  <Step title="กรอกรายละเอียดงาน">
    กรอกข้อมูลในฟิลด์ที่มี:

    | ฟิลด์ | หมายเหตุ |
    |---|---|
    | **Task Name** | ชื่อที่ชัดเจนและกระชับสำหรับงานนี้ |
    | **Description** | บริบท ข้อกำหนด หรือคำแนะนำโดยละเอียด |
    | **Assignee** | สมาชิกทีมที่รับผิดชอบงานนี้ |
    | **Label** | แท็กหมวดหมู่สำหรับจัดระเบียบงาน (เช่น Design, Copy) |
    | **Start Date** | วันที่เริ่มทำงาน |
    | **Due Date** | วันที่ต้องส่งงาน |
    | **Client Review Required** | ช่องทำเครื่องหมายที่กำหนดให้ลูกค้าต้องอนุมัติงานก่อนทำเครื่องหมายว่าเสร็จสิ้น — เปิดใช้งานโดยค่าเริ่มต้น |
    | **Attachments** | ไฟล์อ้างอิงหรือเอกสารประกอบ |
  </Step>

  <Step title="บันทึกงาน">
    คลิก **Save** เพื่อสร้างงาน ลูกค้าจะได้รับการแจ้งเตือนอัตโนมัติว่ามีการเพิ่มงานใหม่ในออเดอร์ของพวกเขา
  </Step>
</Steps>

<Tip>
ช่องทำเครื่องหมาย **Client Review Required** จะเปิดใช้งานโดยค่าเริ่มต้นสำหรับทุกงานใหม่ หากงานใดเป็นงานภายในและไม่ต้องการการอนุมัติจากลูกค้า คุณสามารถยกเลิกการเลือกก่อนบันทึกได้
</Tip>

## สถานะของงาน

งานจะผ่านสถานะต่อไปนี้:

- **In Progress** — กำลังดำเนินการงาน (ไม่ปรากฏให้ลูกค้าหรือพนักงานเห็น)
- **In Review** — ส่งงานเพื่อตรวจสอบแล้ว ลูกค้ามองเห็นได้
- **Done** — งานได้รับการอนุมัติและเสร็จสมบูรณ์แล้ว

<Note>
เมื่อลูกค้าหรือผู้ตรวจสอบปฏิเสธงานจากสถานะ **In Review** สถานะจะเปลี่ยนกลับเป็น **In Progress** โดยอัตโนมัติ
</Note>

## Task Modal และหน้ากิจกรรม

การคลิกที่งานใดก็ตามจะเปิด **Task Modal** ซึ่งเป็นมุมมองรายละเอียดแบบเต็มที่ทำหน้าที่เป็นศูนย์กลางกิจกรรมของงาน จากที่นี่คุณสามารถดูและจัดการ:

- สถานะปัจจุบัน ผู้รับมอบหมาย วันเริ่มต้น และวันครบกำหนด
- คำอธิบายงานและไฟล์แนบต่างๆ
- เธรดฟีดแบ็กและการตอบกลับทั้งหมด
- บันทึกกิจกรรมแบบสมบูรณ์ที่แสดงการอัปเดต ความคิดเห็น และการเปลี่ยนสถานะทุกครั้งของงาน

ทุกคนที่เป็นส่วนหนึ่งของออเดอร์ รวมถึงลูกค้าเมื่อเปิดใช้งานการตรวจสอบโดยลูกค้า สามารถเข้าถึง task modal สำหรับงานที่พวกเขามองเห็นได้

## แก้ไขงาน

หากต้องการอัปเดตรายละเอียดงานหลังจากสร้างแล้ว ให้เปิด task modal และแก้ไขฟิลด์ที่เกี่ยวข้อง (ชื่อ คำอธิบาย ผู้รับมอบหมาย วันที่ ป้ายกำกับ หรือเอกสารแนบ) บันทึกการเปลี่ยนแปลงเพื่อนำไปใช้ เฉพาะ Super Admin, Admin และ PM เท่านั้นที่สามารถแก้ไขงานได้

## ลบงาน

หากต้องการลบงาน ให้เปิดงานและใช้ตัวเลือกลบในเมนูการดำเนินการของงาน การลบเป็นการถาวร บันทึกเวลาและเวอร์ชันไฟล์ที่เกี่ยวข้องทั้งหมดจะถูกลบออกพร้อมกับงาน

<Warning>
การลบงานจะลบเวอร์ชันไฟล์ทั้งหมดและความคิดเห็นฟีดแบ็กที่เกี่ยวข้องด้วย การดำเนินการนี้ไม่สามารถย้อนกลับได้
</Warning>

## การติดตามเวลา

AgencyHandy มีระบบติดตามเวลาในตัวเพื่อให้ทีมของคุณบันทึกชั่วโมงทำงานโดยตรงกับงาน โดยไม่ต้องใช้เครื่องมือภายนอก

### เริ่มจับเวลา

<Steps>
  <Step title="เปิดงาน">
    ไปที่ **Orders → Tasks** และเปิดงานที่ต้องการติดตามเวลา
  </Step>

  <Step title="เริ่มจับเวลา">
    คลิก **Start Timer** ตัวจับเวลาจะเริ่มทำงานและแสดงชื่องาน เวลาที่ผ่านไปแบบเรียลไทม์ และเวลารวมที่บันทึกไว้
  </Step>

  <Step title="หยุดชั่วคราวหรือหยุดเมื่อเสร็จ">
    คลิก **Pause Timer** เพื่อหยุดการติดตามชั่วคราว หรือ **Stop Timer** เพื่อสรุปรายการ เมื่อหยุด คุณจะยืนยันวันที่ เวลาเริ่มต้น เวลาสิ้นสุด ระยะเวลารวม และบันทึกที่ไม่บังคับ
  </Step>
</Steps>

<Note>
คุณสามารถมีตัวจับเวลาที่ทำงานอยู่ได้ครั้งละหนึ่งตัว การเริ่มตัวจับเวลาใหม่จะหยุดตัวที่กำลังทำงานอยู่โดยอัตโนมัติ
</Note>

<Warning>
หากงานถูกทำเครื่องหมายว่า **Done** ตัวจับเวลาจะถูกปิดใช้งาน เปิดงานใหม่เพื่อเปิดใช้งานการบันทึกเวลาอีกครั้ง
</Warning>

### การบันทึกเวลาด้วยตนเอง

หากต้องการบันทึกเวลาย้อนหลัง คลิก **Manual Time Entry** บนงานแล้วกรอก:

- **วันที่**
- **เวลาเริ่มต้นและเวลาสิ้นสุด** (หรือ **ระยะเวลา** เป็นชั่วโมงและนาที)
- **บันทึก** (ไม่บังคับ)

ผู้ใช้หลายคนสามารถบันทึกเวลาในงานเดียวกันพร้อมกันได้ Superadmin และ Admin สามารถสร้างรายการเวลาด้วยตนเองแทนผู้ใช้คนใดก็ได้

### บันทึกการทำงาน

คลิกปุ่ม **Work Log** ข้างตัวจับเวลาเพื่อดูรายละเอียดทั้งหมดของรายการเวลาในงาน รวมถึงวันที่ เวลาเริ่มต้น/สิ้นสุด เวลาทำงานรวม และบันทึกต่างๆ Admin และ Superadmin สามารถลบรายการใดก็ได้ ผู้ใช้ทั่วไปสามารถลบได้เฉพาะรายการของตนเอง

### ไทม์ชีต

ไปที่ **Timesheets** จากแถบด้านซ้ายเพื่อดูมุมมองข้ามออเดอร์ของเวลาที่บันทึกทั้งหมด คุณสามารถกรองตามชื่องาน รหัสออเดอร์ ผู้รับมอบหมาย หรือช่วงวันที่ (รายวัน รายสัปดาห์ รายเดือน) และจัดกลุ่มผลลัพธ์ตามสมาชิกทีม ชื่อบริการ หรือชื่องาน คุณยังสามารถเริ่มจับเวลาหรือเพิ่มรายการด้วยตนเองโดยตรงจากมุมมอง Timesheets ได้

## การรับฟีดแบ็กไฟล์

ระบบรับฟีดแบ็กไฟล์ของ AgencyHandy ช่วยให้คุณและลูกค้าแสดงความคิดเห็นที่แม่นยำบนผลงาน โดยตรงภายในงาน รองรับรูปภาพ PDF วิดีโอ และเว็บไซต์สด พร้อมระบบควบคุมเวอร์ชันและการสนทนาแบบเธรด

### การอัปโหลดไฟล์เพื่อตรวจสอบ

เปิดงาน คลิกไอคอน **➕** ในส่วน **Attachments** แล้วอัปโหลดรูปภาพ PDF วิดีโอ หรือวาง URL เว็บไซต์สด

### การให้ฟีดแบ็กตามประเภทไฟล์

**รูปภาพและ PDF**
เปิดไฟล์และใช้แถบเครื่องมือการเพิ่มคำอธิบาย (Arrow, Line, Rectangle, Highlight) เพื่อทำเครื่องหมายในพื้นที่เฉพาะ เลือกบริเวณ พิมพ์ความคิดเห็น และเลือกว่าจะแชร์กับทีมภายใน ลูกค้า หรือทั้งสองกลุ่ม คุณยังสามารถยกเลิกหรือทำซ้ำการเพิ่มคำอธิบาย และเปลี่ยนสีได้

**วิดีโอ**
เปิดวิดีโอและหยุดที่เฟรมที่ต้องการให้ฟีดแบ็ก พิมพ์ความคิดเห็น วิดีโอจะหยุดโดยอัตโนมัติขณะที่คุณพิมพ์ ฟีดแบ็กจะมีการประทับเวลาของช่วงเวลานั้น คุณสามารถลบการประทับเวลาเพื่อแสดงความคิดเห็นทั่วไปแทนได้

**เว็บไซต์สด**
เพิ่ม URL ของไซต์ในงานโดยใช้ปุ่ม **➕** จากนั้นติดตั้งส่วนขยายเบราว์เซอร์ AgencyHandy เปิดไซต์ใน **View Mode** (โต้ตอบได้) หรือสลับไปที่ **Review Mode** เพื่อถ่ายภาพหน้าจอและเพิ่มคำอธิบายด้วยเครื่องมือวาด ข้อความ หรือรูปร่าง

### การดำเนินการกับความคิดเห็น

ความคิดเห็นแต่ละรายการมีเมนูสามจุดพร้อมตัวเลือกเพื่อ:

- แก้ไขความคิดเห็น
- คัดลอกการตอบกลับ
- ลบความคิดเห็น
- จำกัดการมองเห็นเฉพาะทีมภายในเท่านั้น

### การแก้ไขฟีดแบ็ก

คลิกปุ่ม **✓** บนความคิดเห็นเพื่อทำเครื่องหมายว่า **Resolved** ความคิดเห็นที่แก้ไขแล้วจะยังคงปรากฏให้ดูอ้างอิงและสามารถเปิดใหม่ได้ คุณจะได้รับการแจ้งเตือนเมื่อความคิดเห็นที่คุณทิ้งไว้ถูกแก้ไขโดยคนอื่น

### เวอร์ชันไฟล์

คลิก **Upload New Version** เพื่อเพิ่มไฟล์ที่แก้ไขแล้ว เวอร์ชันก่อนหน้าและฟีดแบ็กของเวอร์ชันเหล่านั้นจะถูกเก็บรักษาไว้และเข้าถึงได้จากเมนูเลื่อนลงของเวอร์ชัน เมื่อเปิดเวอร์ชันเก่า AgencyHandy จะแจ้งเตือนคุณว่ามีเวอร์ชันใหม่กว่า

คุณสามารถระบุป้ายกำกับแต่ละไฟล์เป็น **Needs Review** หรือ **Approved** เพื่อสื่อสารสถานะการตรวจสอบได้อย่างรวดเร็ว

<Note>
ลูกค้าสามารถให้ฟีดแบ็กบนไฟล์ได้ก็ต่อเมื่อเปิดใช้งานช่องทำเครื่องหมาย **Client Review Required** บนงานนั้น การรับฟีดแบ็กไฟล์ใช้ได้เฉพาะภายในงานหรือออเดอร์เท่านั้น ไม่ใช่ในแคตาล็อกบริการหรือรูปภาพโปรไฟล์
</Note>

## งานที่ลูกค้าร้องขอ

ลูกค้าสามารถส่งคำของานจากพอร์ทัลของตน ทำให้มีวิธีที่มีโครงสร้างในการแสดงความต้องการเพิ่มเติมโดยไม่ต้องข้ามกระบวนการอนุมัติของคุณ

### ขั้นตอนสำหรับลูกค้า

<Steps>
  <Step title="เข้าถึงคำขอ">
    ลูกค้าคลิก **Requests** ในแถบด้านซ้ายของพอร์ทัลลูกค้า หรือใช้ปุ่ม **+ Request a Task** โดยตรงภายในออเดอร์
  </Step>

  <Step title="กรอกฟอร์มคำขอ">
    ลูกค้ากรอก:
    - **Order** (บังคับ — คำขอต้องผูกกับออเดอร์ที่มีอยู่)
    - **Task Name** (บังคับ)
    - **Description** (ไม่บังคับ)
    - **Attachments** (ไม่บังคับ — สูงสุด 1 GB รูปแบบรวมถึง PDF, DOCX, JPG, PNG, MP4, ZIP และอื่นๆ)
    - **Start and End Dates** (ไม่บังคับ)
  </Step>

  <Step title="ส่งคำขอ">
    ลูกค้าคลิก **Request** งานจะปรากฏในพอร์ทัลของพวกเขาในสถานะ **Pending** และเอเจนซีของคุณจะได้รับการแจ้งเตือน
  </Step>
</Steps>

### ขั้นตอนสำหรับเอเจนซี

<Steps>
  <Step title="ดูคำขอที่เข้ามา">
    คลิก **Requests** ในแถบด้านซ้ายของพอร์ทัลเอเจนซี คำขอที่รอดำเนินการทั้งหมดจะปรากฏในมุมมองรายการ คุณสามารถกรองตามลูกค้าหรือรหัสออเดอร์ เรียงลำดับตามตัวอักษรหรือวันที่ออก และใช้ **Preferences** เพื่อแสดงหรือซ่อนคอลัมน์
  </Step>

  <Step title="ตรวจสอบคำขอ">
    คลิกคำขอเพื่ออ่านคำอธิบายและดูตัวอย่างเอกสารแนบก่อนตัดสินใจ
  </Step>

  <Step title="อนุมัติหรือปฏิเสธ">
    ใช้ปุ่มดำเนินการข้างแต่ละคำขอเพื่ออนุมัติหรือปฏิเสธทีละรายการ หรือใช้การดำเนินการแบบกลุ่มที่มุมบนขวาเพื่ออนุมัติหรือปฏิเสธคำขอทั้งหมดพร้อมกัน

    - คำขอที่ **อนุมัติ** จะกลายเป็นงานที่ใช้งานได้ในออเดอร์และดำเนินตามขั้นตอนงานปกติ
    - คำขอที่ **ปฏิเสธ** จะยังคงบันทึกไว้พร้อมสถานะ **Rejected** และไม่ถูกเพิ่มลงในออเดอร์
  </Step>
</Steps>

<Note>
ลูกค้าไม่สามารถข้ามขั้นตอนการอนุมัติได้ คำขอต้องการการดำเนินการจากเอเจนซีก่อนที่จะกลายเป็นงานที่ใช้งานได้ คำขอที่รอดำเนินการจะปรากฏให้ทั้งลูกค้าและเอเจนซีเห็นจนกว่าจะมีการตัดสินใจ
</Note>

## มุมมองงานทั้งหมด

นอกเหนือจากการจัดการงานภายในออเดอร์แต่ละรายการ คุณสามารถดูงานทั้งหมดข้ามทุกออเดอร์จากหน้าจอเดียว ไปที่ **Tasks** ในแถบด้านซ้ายเพื่อเข้าถึงหน้า **All Tasks** จากที่นี่คุณสามารถ:

- ดูงานทุกชิ้นที่คุณมองเห็นได้ โดยไม่คำนึงว่าอยู่ในออเดอร์ใด
- กรองและเรียงลำดับตามสถานะ ผู้รับมอบหมาย วันครบกำหนด หรือออเดอร์
- ระบุรายการที่เลยกำหนดหรือมีความเสี่ยงได้อย่างรวดเร็วทั่วทั้งงาน

การมองเห็นข้ามออเดอร์นี้มีประโยชน์อย่างยิ่งสำหรับ Project Manager และ Admin ที่ต้องการปรับสมดุลความสามารถของทีมและติดตามสุขภาพการส่งมอบในระดับเอเจนซี
