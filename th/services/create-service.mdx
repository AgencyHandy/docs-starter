การสร้างบริการใน AgencyHandy เป็นขั้นตอนแรกในการขายข้อเสนอของเอเจนซีของคุณผ่านแพลตฟอร์ม บริการเดียวสามารถมี package ราคาได้สูงสุดห้า package, งานค่าเริ่มต้นอัตโนมัติที่เปิดใช้งานเมื่อมีการสั่งซื้อ, FAQs เพื่อลดคำถามก่อนการขาย, การแนบ portfolio สำหรับหลักฐานทางสังคม และ intake form ที่เป็นตัวเลือก — ทั้งหมดตั้งค่าในที่เดียวก่อนที่คุณจะเผยแพร่

<Note>
  คุณต้องมีบทบาท **Super Admin** หรือ **Admin** เพื่อสร้างบริการ บทบาทอื่นๆ ต้องการสิทธิ์บริการอย่างชัดแจ้งเพื่อเข้าถึงส่วนนี้
</Note>

<Steps>
  <Step title="เลือกประเภทบริการ">
    เปิด **Services** จากแถบด้านข้างซ้ายและคลิก **+ Service** เพื่อเริ่มรายการใหม่

    ที่ด้านบนของหน้า เลือกโมเดลการเรียกเก็บเงินของคุณ:

    - **One-Time Service** — ลูกค้าชำระค่าธรรมเนียมเดียวและรับผลงานที่ส่งมอบครั้งเดียว เหมาะที่สุดสำหรับโปรเจกต์ที่มีขอบเขตชัดเจน: การสร้างเว็บไซต์, แพ็กเกจโลโก้, การตรวจสอบ ฯลฯ
    - **Subscription Service** — การเรียกเก็บเงินจะเกิดซ้ำโดยอัตโนมัติตามตารางที่คุณกำหนด เหมาะที่สุดสำหรับ retainers, เนื้อหาต่อเนื่อง, บริการที่จัดการ หรือแผนการบำรุงรักษา

    <Warning>
      ประเภทบริการจะถูกล็อกหลังจากที่คุณบันทึก หากคุณต้องการโมเดลการเรียกเก็บเงินที่แตกต่างกัน ให้สร้างบริการใหม่
    </Warning>
  </Step>

  <Step title="เพิ่มรายละเอียดบริการ">
    ขยายส่วน **Service Details** และกรอกฟิลด์ต่อไปนี้:

    **Service Name** *(จำเป็น)*
    นี่คือวิธีที่บริการปรากฏให้ลูกค้าเห็นทุกที่ในแพลตฟอร์ม — ในแคตาล็อก, ในใบแจ้งหนี้ และภายใน order workspace ทำให้ชัดเจนและอธิบายได้

    **Media Upload**
    อัปโหลดรูปภาพตัวอย่างหนึ่งรูปหรือมากกว่าเพื่อแสดงบริการในเชิงภาพ รูปภาพคุณภาพสูงเพิ่มความไว้วางใจของลูกค้าและอัตราการแปลง

    **Service Description**
    เขียนคำอธิบายที่ครอบคลุมเกี่ยวกับสิ่งที่บริการรวมถึง, สำหรับใคร และลูกค้าสามารถคาดหวังอะไร ใช้พื้นที่นี้เพื่อสร้างความแตกต่างให้กับข้อเสนอของคุณ

    **Tax Settings**
    เลือกว่าภาษีเป็น **inclusive** (รวมอยู่ในราคาแล้ว) หรือ **exclusive** (เพิ่มบนยอดเมื่อชำระเงิน)

    **Currency**
    กำหนดสกุลเงินสำหรับ packages ทั้งหมดในบริการนี้ packages ทั้งหมดภายในบริการเดียวจะใช้สกุลเงินเดียวกัน

    <Tip>
      อัปโหลดรูปภาพตัวอย่างในอัตราส่วน 16:9 เพื่อการนำเสนอที่ดีที่สุดใน client portal
    </Tip>
  </Step>

  <Step title="สร้าง packages">
    ขยาย **Package Details** และคลิก **+ New Package** เพื่อเพิ่มระดับราคาแรกของคุณ คุณสามารถสร้างได้ **สูงสุด 5 packages** ต่อบริการ

    สำหรับแต่ละ package ตั้งค่าสิ่งต่อไปนี้:

    **Package Name** *(จำเป็น)*
    ชื่อที่ลูกค้าเห็น เช่น *Basic*, *Standard* หรือ *Premium*

    **Package Icon** *(ไม่บังคับ)*
    เลือกไอคอนที่มีในตัวหรืออัปโหลดแบบกำหนดเอง ไอคอนที่อัปโหลดต้องเป็น **32 × 32 pixels**

    **Package Media** *(ไม่บังคับ)*
    อัปโหลดรูปภาพปก specific สำหรับ package นี้ รูปแบบที่รองรับ: PNG, JPG, JPEG ขนาดไฟล์สูงสุด: **5 MB** สูงสุด 1 ไฟล์ต่อ package อัตราส่วนที่แนะนำ: **16:9**

    **Package Brief** *(ไม่บังคับ)*
    สรุปหนึ่งบรรทัดเกี่ยวกับผลงานที่ส่งมอบของ package นี้ สูงสุด **140 อักขระ**

    **What's Included** *(จำเป็น)*
    แสดงรายการผลงานที่ส่งมอบเฉพาะที่รวมอยู่ใน package นี้ คุณสามารถเพิ่มได้ **สูงสุด 15 รายการ** ต่อ package แต่ละรายการควรอธิบายผลงานที่ส่งมอบที่เป็นรูปธรรมหนึ่งอย่าง (เช่น *"บทความบล็อก 5 บทความต่อเดือน ยาวสูงสุด 1,000 คำต่อบทความ"*)

    <Tip>
      หลังจากสร้าง packages หลายอัน ลากเพื่อเรียงลำดับการแสดงให้ลูกค้าเห็น ทำเครื่องหมายอันหนึ่งเป็น **Most Popular** เพื่อดึงความสนใจไปที่ระดับที่คุณแนะนำ
    </Tip>
  </Step>

  <Step title="กำหนดกฎการกำหนดราคาและการส่งมอบ">
    ภายในแต่ละ package กำหนดฟิลด์การกำหนดราคาหลักและการควบคุมที่เป็นตัวเลือก:

    **Amount** *(จำเป็น)*
    ราคาที่ลูกค้าจ่ายสำหรับ package นี้

    **Delivery Time** *(จำเป็น)*
    จำนวนวันโดยประมาณในการส่งมอบบริการหลังจากซื้อ

    **การควบคุมการกำหนดราคาที่เป็นตัวเลือก:**

    <Accordion title="Add Discount">
      ใช้การลดราคาแบบเปอร์เซ็นต์หรือจำนวนเงินคงที่กับราคาของ package นี้ ราคาหลังส่วนลดจะแสดงให้ลูกค้าเห็นควบคู่กับราคาเดิม สร้างผลส่งเสริมการขายที่มีประสิทธิภาพ
    </Accordion>

    <Accordion title="Add Trial Period (subscription only)">
      เสนอทดลองใช้ฟรีหรือราคาลดก่อนที่การเรียกเก็บเงิน subscription ปกติจะเริ่มต้น ตัวเลือกนี้มีให้เฉพาะบริการประเภท subscription เท่านั้น
    </Accordion>

    <Accordion title="Add Quantity Limit">
      จำกัดจำนวนหน่วยที่ลูกค้าสามารถซื้อในรอบการชำระเงินเดียว ใช้สำหรับ packages ที่มีจำนวนจำกัดหรือบริการที่มีข้อจำกัดด้านความจุ
    </Accordion>

    <Accordion title="Add More Price Details">
      แจกแจงค่าใช้จ่ายทั้งหมดเป็นรายการต้นทุนแต่ละรายการ — ตัวอย่างเช่น แสดง *ราคาพื้นฐาน + ค่าธรรมเนียมการตั้งค่า + ค่าใช้จ่ายต่อหน่วย* ป้ายกำกับการแจกแจงแต่ละอันรองรับสูงสุด **20 อักขระ**
    </Accordion>

    <Note>
      ฟิลด์ที่จำเป็นทั้งหมด (Amount และ Delivery Time) ต้องกรอกให้ครบสำหรับทุก package ก่อนที่คุณจะเผยแพร่บริการได้
    </Note>
  </Step>

  <Step title="เพิ่มงานค่าเริ่มต้นสำหรับระบบอัตโนมัติ">
    ทุก package มีส่วน **Default Tasks** งานที่กำหนดไว้ที่นี่จะสร้างขึ้นโดยอัตโนมัติภายใน order workspace ในทันทีที่ลูกค้าซื้อ package นี้ — ไม่จำเป็นต้องตั้งค่าด้วยตนเองหลังจากการขายแต่ละครั้ง

    คลิก **+ Add More** เพื่อเปิด task creation modal สำหรับแต่ละงาน คุณสามารถตั้งค่า:

    | ฟิลด์ | รายละเอียด |
    |---|---|
    | **Task Name** | จำเป็น; สูงสุด **160 อักขระ** |
    | **Description** | คำแนะนำโดยละเอียดสำหรับผู้ได้รับมอบหมาย |
    | **Assignee** | สมาชิกทีมที่รับผิดชอบงานนี้ให้เสร็จ |
    | **Attachment** | อัปโหลดไฟล์อ้างอิง; รูปแบบที่รองรับ: csv, odt, doc, docx, htm, html, pdf, ppt, pptx, txt, xls, xlsx, jpg, jpeg, png, gif, webp, svg, ai, mp4, mp3, wav, zip, rar; ขนาดไฟล์สูงสุด **1 GB** ต่อไฟล์ |
    | **Label** | แท็กงานสำหรับการกรองและรายงาน |
    | **Status** | สถานะเริ่มต้นเมื่อสร้างงาน |
    | **Client Review** | สลับเปิดหากงานนี้ต้องการการอนุมัติจากลูกค้าก่อนจึงจะถือว่าเสร็จสมบูรณ์ |

    เพิ่มงานให้มากเท่าที่จำเป็นเพื่อครอบคลุมกระบวนการส่งมอบสำหรับ package นี้อย่างสมบูรณ์ งานค่าเริ่มต้นที่สอดคล้องกันช่วยให้ทุก order ปฏิบัติตามกระบวนการที่ควบคุมคุณภาพเดียวกัน

    <Tip>
      จับคู่งานค่าเริ่มต้นของคุณกับ SOP ภายในของคุณ ตัวอย่างเช่น package *Social Media Management* อาจสร้างงานอัตโนมัติเช่น *Content Calendar Draft*, *Client Approval*, *Schedule Posts* และ *Monthly Report*
    </Tip>
  </Step>

  <Step title="เพิ่ม FAQs และ portfolio">
    ขยายส่วน **Additional Information** เพื่อเพิ่มสององค์ประกอบที่เป็นตัวเลือกแต่มีผลกระทบสูง:

    **Portfolio Attachment**
    เชื่อมโยงหมวดหมู่ portfolio กับบริการนี้เพื่อให้ลูกค้าสามารถเรียกดูตัวอย่างผลงานในอดีตที่เกี่ยวข้องของคุณได้โดยตรงในรายการบริการ คุณสามารถเชื่อมต่อหมวดหมู่ portfolio ที่มีอยู่หรือสร้างใหม่จากหน้านี้

    **Frequently Asked Questions (FAQs)**
    คลิก **+ Add** เพื่อแทรกคู่คำถามและคำตอบ เพิ่ม FAQs ให้มากเท่าที่เกี่ยวข้อง — หัวข้อทั่วไปรวมถึง:

    - ความเป็นต้นฉบับและสิทธิ์ของเนื้อหา
    - กระบวนการแก้ไขและให้ข้อเสนอแนะ
    - ขอบเขตและสิ่งที่ *ไม่* รวมอยู่
    - ไทม์ไลน์และความถี่ในการสื่อสาร
    - นโยบายการชำระเงินและการคืนเงิน

    FAQs ที่เขียนอย่างดีช่วยลดการโต้ตอบกันก่อนการซื้อและช่วยให้ลูกค้าประเมินตัวเองได้
  </Step>

  <Step title="แนบ intake form">
    Intake form รวบรวมข้อมูลที่คุณต้องการจากลูกค้าก่อนที่งานจะเริ่มต้น — ตัวอย่างเช่น แนวทางแบรนด์, ข้อมูลรับรองการเข้าถึง, กลุ่มเป้าหมาย หรือเป้าหมายโปรเจกต์

    - คลิก **+ Create New Intake Form** เพื่อสร้างแบบฟอร์มตั้งแต่เริ่มต้น หรือเลือกแบบฟอร์มที่มีอยู่จากดรอปดาวน์
    - หากบริการนี้ไม่ต้องการ intake form คลิก **Skip** เพื่อข้ามส่วนนี้
    - คลิก **Update** เพื่อบันทึกการเลือก intake form ของคุณและดำเนินการต่อ

    <Note>
      Intake form เป็นตัวเลือกแต่แนะนำอย่างยิ่งสำหรับบริการที่ต้องการข้อมูลเฉพาะจากลูกค้าก่อนที่การส่งมอบจะเริ่มต้นได้
    </Note>
  </Step>

  <Step title="เผยแพร่บริการ">
    ที่ด้านล่างของตัวแก้ไขบริการ:

    1. สลับ **Publish this Service** ไปยังตำแหน่ง ON
    2. คลิก **Save** (หรือ **Update Service** หากแก้ไขบริการที่มีอยู่)

    เมื่อเผยแพร่แล้ว บริการจะมองเห็นได้โดยลูกค้าใน portal ทันที หากคุณปล่อยให้การสลับปิดอยู่ บริการจะถูกบันทึกเป็นร่างและมองเห็นได้เฉพาะทีมของคุณเท่านั้น

    <Tip>
      ใช้โหมดร่างเพื่อเตรียมบริการล่วงหน้าและเผยแพร่ในเวลาเปิดตัว — มีประโยชน์สำหรับการส่งเสริมการขายตามฤดูกาลหรือการประกาศข้อเสนอใหม่
    </Tip>
  </Step>
</Steps>

## การตั้งค่า Add-On

Add-on คือสิ่งเพิ่มเติมที่เป็นตัวเลือกที่ลูกค้าสามารถเลือกได้ระหว่างการชำระเงิน — ตัวอย่างเช่น *Rush Delivery*, *Extra Revisions* หรือ *Monthly Report* ช่วยให้คุณ upsell บริการเสริมโดยไม่ต้องสร้างรายการใหม่ทั้งหมด

### การสร้าง Add-On

<Steps>
  <Step title="เปิด Add-Ons">
    ไปที่ **Services** ในแถบด้านข้าง คลิกปุ่ม **ลูกศรลง** ข้างหัวข้อ Services และเลือก **Add-Ons**
  </Step>
  <Step title="สร้าง add-on ใหม่">
    คลิก **+ Add New Add-on** และกรอก:

    - **Add-on Name**
    - **Description** — ใช้ rich text editor เพื่ออธิบายอย่างชัดเจนว่า add-on รวมถึงอะไร
    - **Pricing Type** — เลือก *One-Time* หรือ *Subscription*
    - **รายละเอียดการกำหนดราคา:**
      - *One-time:* กรอกจำนวนเงินและสกุลเงิน
      - *Subscription:* กำหนดช่วงเวลาการเรียกเก็บเงิน (เช่น ทุก 1 เดือน), จำกัดรอบการเรียกเก็บเงินที่เป็นตัวเลือก, ค่า subscription และสกุลเงิน
  </Step>
  <Step title="เผยแพร่และบันทึก">
    สลับ **Publish** เป็น ON จากนั้นคลิก **Create** เฉพาะ add-on ที่เผยแพร่เท่านั้นที่จะมองเห็นได้เมื่อเชื่อมต่อกับ order forms
  </Step>
</Steps>

### การแนบ Add-On กับ Order Form

<Steps>
  <Step title="เปิด order form">
    ไปที่ **Forms** และเปิด order form ที่เกี่ยวข้อง
  </Step>
  <Step title="เลือกบริการ">
    เลือกบริการที่คุณต้องการแนบ add-on
  </Step>
  <Step title="เพิ่ม add-ons">
    เลือกส่วน **Add-ons** จากแผงด้านซ้าย จากนั้นเลือก add-on ที่คุณต้องการเชื่อมโยงกับบริการนี้
  </Step>
  <Step title="บันทึกแบบฟอร์ม">
    คลิก **Save** ลูกค้าจะเห็น add-on ที่เลือกเป็นตัวเลือกเพิ่มเติมระหว่างการชำระเงินแล้ว
  </Step>
</Steps>

<Note>
  Add-on ต้องใช้สกุลเงินเดียวกับบริการที่แนบอยู่ด้วย หากสกุลเงินไม่ตรงกัน add-on จะไม่ปรากฏในรายการเลือก
</Note>

### พฤติกรรมการเรียกเก็บเงินของ Subscription Add-On

เมื่อรอบการเรียกเก็บเงินของบริการและ subscription add-on แตกต่างกัน AgencyHandy จะจัดการการทับซ้อนโดยอัตโนมัติ:

<Accordion title="Add-on มีรอบมากกว่าบริการ">
  ทั้งบริการและ add-on จะถูกเรียกเก็บเงินพร้อมกันตลอดระยะเวลาของบริการ หลังจากบริการสิ้นสุด add-on จะยังคงเรียกเก็บเงินต่อไปเพียงอย่างเดียวสำหรับรอบที่เหลือ

  **ตัวอย่าง:** บริการรัน 4 รอบ, add-on รัน 5 รอบ → เรียกเก็บเงินพร้อมกัน 4 รอบ, add-on เรียกเก็บเงินเพียงอย่างเดียวอีก 1 รอบเพิ่มเติม
</Accordion>

<Accordion title="Add-on มีรอบน้อยกว่าบริการ">
  ทั้งบริการและ add-on จะถูกเรียกเก็บเงินพร้อมกันตลอดระยะเวลาของ add-on หลังจาก add-on สิ้นสุด บริการจะยังคงเรียกเก็บเงินต่อไปเพียงอย่างเดียวสำหรับรอบที่เหลือ

  **ตัวอย่าง:** บริการรัน 5 รอบ, add-on รัน 3 รอบ → เรียกเก็บเงินพร้อมกัน 3 รอบ, บริการเรียกเก็บเงินเพียงอย่างเดียวอีก 2 รอบเพิ่มเติม
</Accordion>

## การแก้ไขบริการ

หลังจากที่บริการถูกเผยแพร่แล้ว คุณสามารถอัปเดตฟิลด์ใดๆ ได้โดยไม่รบกวน orders ที่มีอยู่

1. ไปที่ **Services** และค้นหาบริการที่คุณต้องการอัปเดต
2. คลิก **เมนูสามจุด** (⋮) ที่ด้านขวาของแถวบริการ
3. เลือก **Edit** ตัวแก้ไขบริการจะเปิดพร้อมการตั้งค่าที่มีอยู่ทั้งหมด
4. ทำการเปลี่ยนแปลง — อัปเดตชื่อ, คำอธิบาย, การกำหนดราคา, สื่อ, packages, งาน หรือ FAQs
5. คลิก **Update Service** เพื่อบันทึก

<Warning>
  การเปลี่ยนแปลงราคา package หรือเวลาส่งมอบจะมีผลสำหรับ **orders ใหม่เท่านั้น** Orders ที่มีอยู่จะไม่ได้รับผลกระทบย้อนหลัง
</Warning>
