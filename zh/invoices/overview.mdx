AgencyHandy 的发票功能为您的团队提供了一套完整的账单和付款管理系统，集成在您用于管理客户、订单和项目的同一平台中。您可以创建专业发票、通过电子邮件发送给客户、实时跟踪付款状态、为订阅服务自动生成周期账单，并在需要时导出财务记录。每张发票都会自动附带您团队的Logo和地址，确保您的沟通始终保持品牌一致性。

## 谁可以创建发票

只有具备以下角色的用户才能创建和管理发票：

- **超级管理员**
- **管理员**
- **项目经理（PM）**

客户可以查看和下载发票，但不能创建或编辑发票。

## 发票工作台

点击左侧边栏中的 **发票** 以打开发票工作台。在这里您可以：

- 创建、编辑和删除发票
- 按 **客户名称**、**订单 ID** 或 **服务名称** 搜索发票
- 筛选和排序发票列表
- 一目了然地查看发票状态
- 访问发票设置，包括前缀配置

## 发票状态

每张发票都遵循既定的生命周期。理解这些状态有助于您有效管理应收账款。

| 状态 | 说明 |
|--------|-------------|
| **草稿** | 发票已创建但尚未完成。您可以编辑其中任何内容。准备好后，通过将状态更改为"待付款"来完成发票。未关联订阅的草稿发票可以删除。 |
| **待付款/未付款** | 发票已完成，等待付款。团队将其发送给客户，客户可以查看并付款。您可以将待付款发票改为已付款、已作废或无法收回。 |
| **已付款** | 发票已付款。将发票标记为已付款时，系统会自动记录付款日期。 |
| **已作废** | 发票已取消。系统自动记录作废日期。 |
| **无法收回** | 客户不太可能付款。在您的会计流程中将此视为坏账处理。 |

<Note>
  当您使用 **发送给客户** 按钮将发票发送给客户时，状态将自动从 **草稿** 变为 **待付款**。只有团队才能将状态从待付款更改为已付款、已作废或无法收回。
</Note>

## 直接发票（无需订单 ID）

AgencyHandy 支持 **直接发票**——不关联任何现有订单的发票。这适用于一次性账单、临时付款，或向尚未在您系统中注册为客户的联系人开票。

直接发票的主要功能：

- 无需选择订单 ID 即可发送发票
- 向任意现有客户开票，或使用 **独立邮件** 向任意电子邮件地址发送
- 手动添加项目、税费和折扣
- 立即发送或保存为草稿

<Tip>
  当您需要为没有正式订单的工作快速向客户收费时，请使用直接发票——例如加急项目、咨询电话或超出范围的工作。
</Tip>

## 导出发票

您可以导出发票列表，用于外部会计工具或存档。导出内容包含所有可见列：发票 ID、客户名称、服务、金额、状态和日期。可通过发票工作台访问导出选项。

## 发票设置

在发票工作台右上角的 **设置** 按钮中配置全局发票设置，包括：

- **发票前缀** — 自定义每个发票编号前显示的前缀
- **提醒模板** — 为待付款发票设置自动付款提醒
